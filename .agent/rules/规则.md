### 编程规则：全链路中文工作流

1. 核心语言要求

所有的交互、推理、计划和文档必须完全使用 简体中文。

除非是代码中的特定变量、库名或通用的行业术语（如 Props, Component, API 等），否则严禁使用英文。

2. 交付物（Artifacts）标准

Task Lists (任务列表)：每一项任务的描述必须是中文，清晰指明操作目标。

Implementation Plans (执行方案)：架构设计、逻辑流程、步骤拆解必须使用中文撰写，确保方案易于理解。

Walkthroughs (代码演示)：在完成修改后，必须用中文总结变更点、测试步骤和验证结果。

3. 代码注释与文档

所有的代码逻辑保持标准编程语法。

核心要求：所有新生成的或修改的 代码注释 (Comments)、JSDoc/Docstrings 以及项目文档必须使用中文。

4. 术语处理

遇到关键技术术语，首选“中文名称 (英文名称)”的格式，例如：“中间件 (Middleware)”。

5. 技术栈与架构规范

- **Next.js 版本**：项目固定使用 **Next.js 16**。
- **异步 API 处理**：必须遵循 Next.js 16 的规范，所有 `params`、`searchParams`、`cookies()`、`headers()` 等 API 必须作为 **Promise** 处理，使用 `await` 或 React 的 `use()` 进行解构。严禁同步访问这些属性。

6. 环境规范

- **终端要求**：Windows 环境下必须使用 PowerShell 7+ (`pwsh`) 作为默认终端，禁止使用旧版 `powershell` (v5.x)。所有脚本和命令必须兼容 `pwsh`。
