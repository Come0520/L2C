# Showroom (云展厅) 模块 L5 升级任务

## 任务目标
将 `showroom` 模块从 **L4 (6.5)** 提升至 **L5**。目前该模块功能完善且测试覆盖率高达 74.5%，代码也没有 `any`。但在**可运维性（日志极少）**、**文档完整性**与**UI/UX**等方面存在提升空间。

## 当前短板与改进方向

### 1. D7: 可运维性 (当前: 5分)
- **现状**：整个模块只有 2 个日志，7 次审计调用。
- **任务**：
  - 在相关的 actions（如展品管理、分享链接生成、积分操作）中增加必要的 `logger.info` 和 `logger.error`。
  - 增加必要的 `try-catch` 捕获并将底层报错转为安全的用户提示。

### 2. D5: UI/UX 成熟度 (当前: 5分)
- **现状**：没有设立专门的 components 目录，视图逻辑可能堆积在路由中。
- **任务**：
  - 如果视图层有重复利用的结构，请提取到 `src/features/showroom/components/` 下。
  - 改善列表和详情页的 Loading 状态和 Empty 态。

### 3. D4: 文档完整性 (当前: 6分)
- **现状**：约 30 个 JSDoc，但可能不够全面。
- **任务**：
  - 对核心的 Actions 和接口添加详细的 **中文 JSDoc 注释**。

## 执行要求
1. **语言**：所有代码注释、日志信息、UI文案必须全中文。
2. **测试维护**：确保优化后的模块继续通过所有涉及的单元和集成测试，**测试覆盖率不得下降**。
3. **类型安全**：保持 0 `any` 和完美的 TypeScript 类型验证。不要引入任何 `@ts-expect-error`。
4. **日志**：请从 `@/shared/lib/logger` 引入日志。
