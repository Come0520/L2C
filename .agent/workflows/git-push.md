# 📖 Git 基础概念小课堂

为了方便您理解，这里简单解释一下核心概念：

## 1. Main 和 Branch (分支) 的区别
想象您正在写一本小说：
*   **Main (主分支)**：这是**“正式出版”**的版本。这里的代码应该是最稳定、随时可以展示给别人看的。
*   **Branch (分支)**：这是您的**“草稿纸”**或**“平行宇宙”**。
    *   当您想开发一个新功能（比如给主角加个技能）或者修一个 Bug 时，通常会从 Main 复制一份出来（切分支），在草稿上随意涂改。
    *   只有当您确信草稿改得很完美了，才会把它**合并 (Merge)** 回 Main。
    *   **当前策略**：为了简便，我们目前暂时直接在 **Main** 上工作（单人快速迭代模式）。

## 2. 版本号与更迭
*   **Commit (提交)**：这就是一个**“游戏存档”**。每次您完成了一小块工作（比如“做好了登录页”），我们就存个档。这样如果以后搞砸了，随时可以回档。
*   **Push (推送)**：把您电脑上的存档，上传到云端 (GitHub) 备份。
*   **版本号 (如 v1.0.1)**：
    *   这只是给某个特殊的存档贴个显眼的标签，比如 v1.0 代表“第一版正式发布”。
    *   平时开发中，我们更多关注的是 Commit Message（存档备注），只要备注写得清楚（比如“修复了登录按钮的颜色”），就能知道目前的进展。

# 🚀 自动化提交流程

1. **查看当前状态**
   *   查看有哪些文件被修改了。
   `git status`

2. **添加所有更改 (存入暂存区)**
   *   把所有改动的文件放到“待存档”的盒子里。
   // turbo
   `git add .`

3. **提交更改 (正式存档)**
   *   **Agent 注意**: 请根据 `git status` 的变更内容，自动生成一句清晰的**中文**描述，解释这次做了什么。
   `git commit -m "具体的变更描述"`

4. **推送到远程仓库 (上传云端)**
   *   把本地的存档同步到 GitHub。
   // turbo
   `git push`
