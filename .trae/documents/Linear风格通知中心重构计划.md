# Linear风格通知中心重构计划

## 1. 核心目标
保持原有业务逻辑不变，将通知中心页面重构为符合Linear风格的现代SaaS设计，解决当前的风格割裂问题。

## 2. 重构范围
- **文件**: `/src/app/dashboard/notifications/page.tsx`
- **业务逻辑保留**: TypeScript接口、Mock数据、过滤逻辑、搜索功能
- **UI/UX重构**: 视觉风格、组件库、动画效果、交互体验

## 3. 重构步骤

### 3.1 环境准备
- 确保已安装 `framer-motion` 和 `lucide-react` 依赖
- 确保主题变量（`bg-theme-bg`等）已在项目中配置

### 3.2 组件重构
1. **移除Paper组件**: 将所有PaperCard、PaperButton等替换为符合Linear风格的自定义组件或原生HTML+Tailwind
2. **封装独立组件**: 
   - `NotificationItem`: 通知列表项
   - `ApprovalItem`: 审批列表项
   - 移除重复的switch语句，优化代码结构

### 3.3 视觉风格调整
1. **深色模式适配**: 使用`bg-theme-bg`、`text-theme-text-primary`等变量
2. **移除Emoji**: 替换为`lucide-react`图标库
3. **卡片设计**: 采用`bg-theme-bg-secondary`、`border-theme-border`等样式
4. **排版优化**: 调整字体大小、行高、间距，提升可读性

### 3.4 交互体验提升
1. **添加Framer Motion动画**: 
   - Tab切换动画（使用layoutId）
   - 列表项进入/退出动画
   - Hover效果动画
2. **Timeline设计**: 将审批流程改为垂直时间轴样式
3. **搜索框优化**: 增加搜索图标，提升视觉效果
4. **状态指示器**: 优化未读状态、审批状态的视觉表现

### 3.5 代码优化
1. **使用cn工具**: 优化类名组合
2. **简化条件判断**: 使用对象映射替代频繁的switch语句
3. **提高代码复用性**: 提取公共样式和逻辑
4. **优化JSX结构**: 减少嵌套，提升可读性

## 4. 重构后的功能保持
- 通知列表展示
- 审批列表展示
- 搜索功能
- 过滤功能
- 详情查看
- 审批操作

## 5. 预期效果
- 与Dashboard页面风格统一，无割裂感
- 现代科技感的UI设计
- 流畅的动画交互
- 清晰的信息层级
- 优化的用户体验

## 6. 实施时间
预计1-2小时完成重构和测试
