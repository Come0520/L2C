# Next.js 15 升级计划

## 1. 依赖检查与更新

### 1.1 核心依赖
- 确保 `next` 版本为 `^15.0.0`（已完成）
- 确保 `react` 和 `react-dom` 版本为 `^19.0.0`（已完成）
- 确保 `typescript` 版本为 `^5.0.0`（已完成）

### 1.2 相关依赖
- 更新 `eslint-config-next` 到 `^15.0.0`（已完成）
- 检查并更新其他依赖项，确保与 Next.js 15 兼容

## 2. 配置文件更新

### 2.1 next.config.js
- 检查并更新配置项，确保使用 Next.js 15 的最新配置语法
- 移除任何 Next.js 14 特定的配置

### 2.2 TypeScript 配置
- 确保 `tsconfig.json` 符合 Next.js 15 的要求
- 更新 `next-env.d.ts`（如果需要）

## 3. 代码更新

### 3.1 App Router 最佳实践
- 确保所有页面和组件都使用 App Router 语法
- 检查并更新 `useRouter`、`useParams` 等钩子的使用
- 确保 `Image` 组件使用正确的导入路径和属性

### 3.2 服务器组件
- 确保服务器组件和客户端组件的正确使用
- 更新 `'use client'` 指令的使用

### 3.3 API 路由
- 确保 API 路由使用 Next.js 15 的最新语法
- 检查并更新 `route.ts` 文件

## 4. 测试与验证

### 4.1 运行开发服务器
- 确保 `npm run dev` 能够正常启动
- 检查是否有任何警告或错误

### 4.2 运行构建命令
- 确保 `npm run build` 能够成功完成
- 检查是否有任何构建错误或警告

### 4.3 运行类型检查
- 确保 `npm run typecheck` 能够通过
- 修复任何类型错误

### 4.4 运行测试
- 确保 `npm run test` 能够通过
- 修复任何测试失败

## 5. 清理工作

### 5.1 移除过时的代码
- 移除任何与 Next.js 14 相关的注释或文档
- 清理任何不再使用的导入或依赖

### 5.2 更新文档
- 更新项目文档，确保使用 Next.js 15 的相关描述
- 更新 README.md（如果需要）

## 6. 验证部署

- 确保项目能够正常部署到生产环境
- 检查生产环境是否有任何问题

## 升级优先级

1. 依赖更新（必须）
2. 配置文件更新（必须）
3. 代码更新（必须）
4. 测试与验证（必须）
5. 清理工作（建议）
6. 更新文档（建议）
7. 验证部署（必须）

## 预期结果

- 项目完全适配 Next.js 15
- 没有任何与 Next.js 14 相关的内容
- 项目能够正常开发、构建和部署
- 充分利用 Next.js 15 的新特性和最佳实践
