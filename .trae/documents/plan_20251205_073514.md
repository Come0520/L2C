# 脚手架与模板实现计划

## 目标
为 `features/*` 与 `services/*` 提供文件模板，通过脚手架工具一键生成标准模块结构，提高开发效率，保证代码风格一致性。

## 实施步骤

### 1. 安装依赖
- 安装 Plop 作为脚手架工具
- 添加到开发依赖中

### 2. 创建 Plop 配置
- 在项目根目录创建 `plopfile.js` 配置文件
- 配置生成器：
  - `feature` 生成器：用于生成 features 模块
  - `service` 生成器：用于生成 services 模块

### 3. 创建模板文件

#### Features 模块模板
- 基础目录结构
- 组件模板
- 测试文件模板
- 类型定义模板

#### Services 模块模板
- 服务客户端模板
- 测试文件模板

### 4. 添加 npm 脚本
- 在 `package.json` 中添加 `generate` 脚本，用于启动 Plop
- 支持直接指定生成器和名称：`npm run generate feature -- --name=my-feature`

### 5. 测试生成功能
- 测试生成 features 模块
- 测试生成 services 模块
- 验证生成的文件结构是否符合项目规范

## 预期结果
- 开发者可以通过 `npm run generate` 命令交互式生成模块
- 生成的模块结构符合项目规范
- 包含基础的测试文件
- 提高开发效率，保证代码风格一致性

## 技术栈
- Plop：脚手架工具
- Handlebars：模板引擎（Plop 内置）
