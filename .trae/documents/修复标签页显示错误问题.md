## 问题分析

1. **现象**：当访问订单处理阶段的页面（如 `/orders/status/draft-sign` 或 `/orders/status/installing-pending-assignment`）时，顶部标签页仍然显示工作台的标签页。

2. **根本原因**：在 `dashboard-layout.tsx` 文件中，`activeGroup` 的计算逻辑存在问题：
   ```typescript
   const activeGroup = React.useMemo(() => {
     const group = navGroups.find(group =>
       group.items.some(item => pathname === item.href || pathname?.startsWith(item.href))
     );
     return group || navGroups[0];
   }, [pathname, navGroups]);
   ```

3. **具体原因**：
   - 当访问 `/orders/status/draft-sign` 时，
   - 工作台组（`navGroups[0]`）的 items 中包含 `/orders` 路径，
   - 由于 `pathname?.startsWith(item.href)` 条件，`/orders/status/draft-sign` 会匹配 `/orders`，
   - 因此 `activeGroup` 被错误地设置为工作台组。

## 修复方案

修改 `activeGroup` 的计算逻辑，使其更精确地匹配路径：

1. **优化匹配逻辑**：优先检查精确匹配，然后检查以 `item.href/` 开头（带斜杠）的路径
2. **按匹配长度排序**：确保最长的匹配项被选中，避免短路径匹配长路径

## 代码修改

```typescript
// Find current active group
const activeGroup = React.useMemo(() => {
  // First try to find exact match or match with trailing slash
  const exactMatchGroup = navGroups.find(group =>
    group.items.some(item => pathname === item.href || pathname?.startsWith(`${item.href}/`))
  );
  
  if (exactMatchGroup) return exactMatchGroup;
  
  // If no exact match, find the group with the longest matching prefix
  let bestMatch: typeof navGroups[0] | null = null;
  let maxMatchLength = 0;
  
  navGroups.forEach(group => {
    group.items.forEach(item => {
      if (pathname?.startsWith(item.href) && item.href.length > maxMatchLength) {
        bestMatch = group;
        maxMatchLength = item.href.length;
      }
    });
  });
  
  return bestMatch || navGroups[0];
}, [pathname, navGroups]);
```
