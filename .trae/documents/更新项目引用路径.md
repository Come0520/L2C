# 更新项目引用路径

## 问题分析
根据最新的目录结构，我们之前进行了项目结构优化，将组件、类型和工具函数进行了重新组织，但项目中仍然存在大量引用旧路径的文件，导致构建失败。

## 需要更新的路径

### 1. 组件路径
- **旧路径**：`@/components/`
- **新路径**：`@/components/client-components/`
- **影响范围**：所有引用组件的页面和组件文件

### 2. lib/utils 路径
- **旧路径**：`@/lib/utils`
- **新路径**：`@/utils/lib-utils`
- **影响范围**：所有使用了 cn 函数和其他工具函数的组件文件

## 解决方案

### 1. 更新组件引用路径
使用 grep 和 sed 命令批量替换所有组件引用路径：
```bash
grep -r "@/components/" --include="*.ts" --include="*.tsx" src/ | xargs sed -i '' 's|@/components/|@/components/client-components/|g'
```

### 2. 更新 lib/utils 引用路径
使用 grep 和 sed 命令批量替换所有 lib/utils 引用路径：
```bash
grep -r "@/lib/utils" --include="*.ts" --include="*.tsx" src/ | xargs sed -i '' 's|@/lib/utils|@/utils/lib-utils|g'
```

### 3. 验证更新结果
运行 lint 和 typecheck 命令确认所有路径都已正确更新：
```bash
npm run lint
npm run typecheck
```

## 预期结果
- 所有组件引用路径都已更新为 `@/components/client-components/`
- 所有 lib/utils 引用路径都已更新为 `@/utils/lib-utils`
- 项目可以成功通过 lint 和 typecheck 检查
- 项目可以正常构建和运行

## 注意事项
- 在执行批量替换命令前，建议先备份项目文件
- 替换完成后，需要仔细检查替换结果，确保没有误替换
- 对于特殊情况，可能需要手动更新路径

## 后续建议
- 建立代码审查机制，确保新代码使用正确的路径
- 考虑使用 TypeScript 路径映射，简化路径管理
- 定期检查和清理项目中的无效引用
