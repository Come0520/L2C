# 统一状态流程图计划

## 1. 目标

统一状态流程图中的状态名称、角色定义、触发条件和时效要求，确保整个文档的一致性。

## 2. 范围

- 正常流程流转条件（第356-372行）

## 3. 统一标准

### 3.1 状态名称

使用我们之前更新的状态名称，包括：

| 阶段 | 状态名称 |
|------|----------|
| 线索阶段 | 线索分配、安排测量 |
| 测量阶段 | 分配测量师、确认测量 |
| 订单阶段 | 推单处理、审核下单、生成采购单、整单发货、填写快递单号 |
| 安装阶段 | 安排安装、分配安装师、确认安装、上传安装图片、审核安装结果 |
| 财务阶段 | 财务对账、开具发票、确认回款 |
| 通用操作 | 取消订单、修改备注 |

### 3.2 角色定义

使用我们的角色定义，包括：

| 角色 | 描述 |
|------|------|
| 销售人员 | 负责线索跟踪、方案确认、推单等 |
| 派单员 | 负责分配测量师和安装师 |
| 测量师 | 负责上门测量 |
| 安装师 | 负责上门安装 |
| 订单客服 | 负责审核下单、填写快递单号、生成采购单等 |
| 财务 | 负责对账、开票、确认回款等 |
| 销售负责人 | 负责销售团队管理 |
| 渠道负责人 | 负责渠道管理 |
| 领导 | 负责审批等高级权限 |
| 客户 | 负责确认方案、查看订单等 |
| 业务 | 负责线索分配等 |
| 管理员 | 拥有所有权限 |

### 3.3 触发条件

根据我们之前更新的业务规则，修正触发条件：

| 状态流转 | 触发条件 | 执行角色 |
|---------|----------|----------|
| 待下单 → 生产中 | 订单客服审核通过后进入 | 订单客服 |
| 生产中 → 备货完成 | 当所有采购单都进入备货完成状态后自动进入 | 系统 |
| 备货完成 → 待发货 | 销售人员点击"整单发货"后进入 | 销售人员 |
| 待发货 → 安装中-待分配 | 当所有采购单号都有了对应的快递单号后自动进入 | 系统 |
| 推单处理 | 销售人员进行推单操作 | 销售人员 |
| 审核下单 | 订单客服审核上传的HOME推单和销售单 | 订单客服 |
| 生成采购单 | 订单客服支持多选商品生成一条采购单 | 订单客服 |
| 填写快递单号 | 订单客服为每一个采购单填写对应的快递单号 | 订单客服 |

### 3.4 时效要求

根据我们之前更新的时效要求，修正各环节时效：

| 状态 | 时效要求 | 执行角色 |
|------|----------|----------|
| 生产中 | 根据产品类型设定（窗帘7天、轨道3天、墙咔15天、墙布7天） | 订单客服 |
| 备货完成 | 24小时内必须点击"整单发货" | 销售人员 |
| 待发货 | 24小时内必须填写所有快递单号 | 订单客服 |
| 安装中-待分配 | 4小时内必须分配安装师 | 派单员 |
| 安装中-分配中 | 2小时内必须接单或拒绝接单 | 安装师 |
| 安装中-待上门 | 7天内必须完成上门安装 | 安装师 |
| 安装中-待确认 | 24小时内必须完成安装图片上传和截图上传 | 销售人员 |
| 待对账 | 3个工作日内必须完成对账 | 财务 |
| 待开发票 | 3个工作日内必须完成开票 | 财务 |
| 待回款 | 30天内必须完成回款确认 | 财务 |
