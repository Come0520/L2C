## 实现计划

### 1. 创建待开票页面组件
- 创建 `pending-invoice-view.tsx` 组件文件
- 定义待开票订单的数据类型，包含对账单号、城市、金额等字段
- 实现模拟数据用于展示

### 2. 实现对账单列表
- 使用现有的 `PaperTable` 组件构建列表
- 包含字段：对账单号、城市、金额、上传发票、确认
- 实现搜索和筛选功能

### 3. 实现发票上传功能
- 使用现有的 `PaperFileUpload` 组件
- 限制只能上传 PDF 格式文件
- 显示已上传的文件名

### 4. 实现审核逻辑
- 上传发票后，确认按钮才可用
- 实现确认功能，更新订单状态
- 实现驳回功能，允许填写驳回原因
- 添加 Toast 通知反馈操作结果

### 5. 集成到状态页面
- 在 `[status]/page.tsx` 中导入新组件
- 添加 `pending-invoice` 状态的路由处理

### 6. 样式和交互优化
- 保持与现有页面一致的设计风格
- 添加适当的动画和过渡效果
- 确保响应式设计

## 技术要点

- 使用 React Hooks 管理状态
- 遵循现有代码库的组件结构和命名规范
- 实现表单验证和错误处理
- 保持代码简洁和可维护性
