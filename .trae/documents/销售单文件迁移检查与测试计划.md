## 销售单文件迁移检查与测试计划

### 一、路径引用检查
1. **类型检查**：运行 `pnpm typecheck` 验证所有导入路径是否正确
2. **别名配置验证**：确认 `@/*` 别名正确指向 `./src/*`
3. **组件导入检查**：验证销售单组件间的导入关系是否正确
4. **服务层导入检查**：验证销售单服务的导入路径是否正确
5. **钩子导入检查**：验证 `useSalesOrders` 等钩子的导入路径是否正确

### 二、流程测试
1. **构建测试**：运行 `pnpm build` 确保项目能够成功构建
2. **单元测试**：运行销售单相关的单元测试，重点检查：
   - `pnpm test src/hooks/__tests__/useSalesOrders.test.tsx`
   - `pnpm test src/app/__tests__/orders.test.tsx`
3. **lint 检查**：运行 `pnpm lint` 确保代码符合规范
4. **开发环境测试**：启动开发服务器验证功能是否正常

### 三、关键文件验证
1. **销售单列表组件**：`sales-order-list.tsx`
2. **销售单详情组件**：`sales-order-detail.tsx`
3. **销售单表单组件**：`sales-order-form.tsx`
4. **订单表单钩子**：`useOrderForm.ts`
5. **销售单服务**：`salesOrders.client.ts`
6. **销售单钩子**：`useSalesOrders.ts`

### 四、执行步骤
1. 运行类型检查
2. 运行 lint 检查
3. 运行单元测试
4. 运行构建测试
5. （可选）启动开发服务器验证功能

通过以上步骤，可以确保销售单文件迁移后的路径引用正确，并且流程通顺。
