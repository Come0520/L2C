# 罗莱L2C销售管理系统 - 业务需求分析（精简版）

## 📈 市场机会
- **市场规模**：中国家居软装市场超2万亿，年增15%+
- **数字化痛点**：90%企业仍用纸质流程，效率低下
- **政策驱动**：国家推进产业数字化转型

## 🏗️ 技术架构
- **MVP阶段**：Next.js 15+ + Supabase BaaS，快速验证
- **扩展阶段**：Edge Functions + Realtime，优化性能
- **成熟期**：全Serverless架构，事件驱动

## 👥 核心用户群体

### 1. 销售团队（主要用户）
- **销售负责人**（10%）：需要全局数据视图和决策支持
- **驻店销售**（40%）：需要订单跟踪和客户管理工具
- **远程销售**（30%）：负责推单处理，上传采购申请截图
- **订单客服**（20%）：负责订单跟踪和物流管理

### 2. 服务提供商（关键用户）
- **测量师**（35%）：需要智能派单和技能匹配
- **安装师**（55%）：需要详细任务信息和质量反馈
- **派单员**（10%）：需要智能调度工具

### 3. 管理层（决策用户）
- **销售领导**：需要实时数据和审批权限
- **财务专员**：需要对账和收入确认功能
- **审批员**：需要跨门店协调工具

## 🏢 核心功能需求

### 1. 线索管理
- **线索单**：独立线索管理，5状态流转
- Excel批量导入（1000+条）
- 智能重复检查
- 自动分配规则
- 7/15/30天预警机制
- **预报价**：线索阶段Excel报价文件（支持.xlsx/.xls格式）

### 2. 核心单据体系
- **销售单**：27状态完整流转（线索→成交全流程）
- **测量单**：智能派单→测量确认（第二阶段开发）
- **安装单**：任务调度→质量验收（第二阶段开发）

### 3. 积分激励机制
- 多节点发放（创建/跟进/转化/成交）
- 在途积分管理（保护公司权益）
- 积分商城兑换

### 罗莱好伙伴管理
- **角色**：圣都设计师+导购（每单最多2人）
- **协作**：草签动作提供专业支持（无订单操作权限）
- **激励**：基于订单金额的积分分配
- **延迟确认**：7天机制确保准确性

### 5. 财务对账
- **对账单**：销售数据与财务数据自动对账
- 多节点回款跟踪（定金/进度款/尾款）
- 收入确认管理
- 发票流程管理

## 💰 商业模式

### 收入模式
- **SaaS订阅**：50-150元/用户/月
- **按订单量**：免费版100单/月→高级版不限单量
- **增值服务**：积分商城抽成5%-10%

### 成本结构
- 技术开发（40%）：云服务+第三方服务
- 运营维护（30%）：客服+市场推广
- 管理成本（30%）：团队+办公

## 🔍 关键业务流程

### 草签动作
- **性质**：业务动作（非状态），线索→订单的关键节点
- **结果**：订单进入"预算中"状态
- **文档**：上传《预算报价单》后进入"等待推单"
- **权限**：驻店销售主导，外部伙伴协助

### 推单流程
- **定义**：圣都采购申请→罗莱推单确认
- **操作**：远程销售上传截图→订单客服确认
- **时效**：48小时内完成

### 状态流转核心
待分配→待跟踪→跟踪中→草签→待测量→测量中→方案待确认→待推单→待下单→生产中→备货完成→安装中→待对账→待回款→已完成
