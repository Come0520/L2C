# 罗莱L2C销售管理系统 - 业务需求分析

## 📈 市场分析

### 家居软装行业现状
**市场规模**：中国家居软装市场规模超过2万亿元，年增长率15%+
**数字化程度**：行业整体数字化水平较低，多数企业仍采用传统纸质流程
**竞争格局**：市场分散，缺乏统一的数字化服务平台

### 数字化转型趋势
**政策驱动**：国家大力推进产业数字化转型
**需求升级**：消费者对服务透明度和效率要求提升
**技术成熟**：云计算、大数据、AI技术日趋成熟，成本降低

## 🏗️ 技术架构演进路径

### MVP阶段：Next.js + Supabase BaaS架构
采用Next.js + Supabase BaaS架构快速实现MVP功能，利用Serverless和BaaS能力降低运维成本：
- **前端**：Next.js 15+（App Router, Server Components, Server Actions）
- **后端**：Supabase BaaS（Auth, Database, Realtime, Storage, Edge Functions）
- **数据库**：PostgreSQL（Supabase托管）
- **部署**：Vercel + Supabase Cloud，简化运维管理

### 扩展阶段：Serverless优化
根据业务需求优化Serverless架构：
- **Edge Functions**：处理复杂业务逻辑和第三方API集成
- **实时数据同步**：利用Supabase Realtime实现数据实时更新
- **细粒度RLS策略**：优化Row Level Security策略，提升性能和安全性
- **监控体系**：完善监控、日志、链路追踪系统

### 成熟期：全Serverless架构
形成完整的全Serverless架构，支持独立扩展和部署：
- **事件驱动**：基于事件溯源的最终一致性架构
- **边缘计算**：利用Edge Functions实现低延迟处理
- **全球CDN加速**：提升全球用户访问速度
- **自动扩缩容**：根据业务负载自动调整资源

## 👥 用户画像分析

### 核心用户群体

#### 1. 销售团队（主要用户）
**销售负责人（10%）**
- 年龄：30-45岁
- 特征：销售管理经验，全面负责门店或地区销售
- 痛点：缺乏全面数据视图，决策依据不足
- 需求：全局销售数据，团队绩效管理，决策支持
- 权限：门店或地区所有销售相关权限

**驻店销售（40%）**
- 年龄：25-35岁
- 特征：门店销售经验，熟悉客户沟通
- 痛点：订单状态不透明，客户询问频繁，缺乏有效工具
- 需求：实时订单跟踪，客户管理工具，销售激励

**远程销售（30%）**
- 年龄：28-40岁
- 特征：专业销售背景，商务谈判能力强
- 痛点：跨部门协调困难，订单执行效率低
- 需求：订单全流程管理，智能提醒，数据分析
- 职责：协助进群、报价、对接拍单员回传测量数据报价，负责推单处理，上传圣都Home系统采购申请截图
- 备注：可与驻店销售为同一人，一个账户可同时赋予两种身份，若同一人则由该人员负责推单处理

**订单客服（20%）**
- 年龄：25-35岁
- 特征：订单处理经验，协调能力强
- 痛点：手工操作多，容易出错，效率低下
- 需求：自动化工具，批量处理，系统集成
- 职责：主要负责后期订单跟踪和所有物流信息，将订单下到帘客云或其他下单软件
- 备注：属于交付团队，不再归类为销售团队

#### 2. 服务提供商（关键用户）
**测量师（35%）**
- 年龄：30-50岁
- 特征：技术工种，专业技能强
- 痛点：任务分配不均，沟通成本高
- 需求：智能派单，技能匹配，收入透明

**安装师（55%）**
- 年龄：25-45岁
- 特征：安装经验丰富，服务意识强
- 痛点：任务信息不完整，客户沟通困难
- 需求：详细任务信息，客户联系方式，质量反馈

**派单员（10%）**
- 年龄：28-45岁
- 特征：调度管理经验，协调能力强
- 痛点：手工派单效率低，资源分配困难
- 需求：智能调度工具，服务商管理，绩效监控

#### 3. 管理层（决策用户）
**销售领导（40%）**
- 年龄：35-50岁
- 特征：销售管理经验丰富，具备决策权限
- 痛点：缺乏实时数据，决策滞后，审批流程不透明
- 需求：销售数据分析，团队绩效管理，预警机制，审批权限

**财务专员（30%）**
- 年龄：30-45岁
- 特征：财务专业背景，熟悉企业财务流程
- 痛点：销售数据与财务数据不同步，对账困难，收入确认滞后
- 需求：销售数据实时同步，自动对账功能，收入确认管理，财务报表生成

**审批员（20%）**
- 年龄：32-48岁
- 特征：总部协调经验，跨部门沟通能力强，熟悉全国门店运营
- 痛点：门店间协调困难，审批流程不统一，信息传递效率低
- 需求：统一审批平台，门店协调工具，标准化流程，信息实时同步

**系统管理员（10%）**
- 年龄：28-45岁
- 特征：IT技术背景，系统管理经验
- 痛点：系统维护复杂，数据分散
- 需求：统一管理平台，自动化运维，数据备份

### 延伸用户群体

#### 4. **外部合作伙伴（协同用户）**
**圣都设计师（50%）**
- 年龄：28-45岁
- 特征：专业设计背景，熟悉家居软装行业
- 痛点：客户资源获取困难，设计成果转化率低
- 需求：客户资源共享，设计成果展示，积分激励

**圣都导购（50%）**
- 年龄：25-40岁
- 特征：门店销售经验，熟悉产品特性
- 痛点：客户跟进困难，销售工具不足
- 需求：客户管理工具，产品知识库，销售辅助

## 🏢 客户需求分析

### 功能性需求

#### 核心业务需求
1. **线索管理**
   - Excel批量导入，支持1000+条记录
   - 智能重复检查，相似度算法匹配
   - 自动分配规则，负载均衡算法
   - 预警提醒机制，7/15/30天分级预警

2. **三单体系管理**
   - 销售单27状态完整流转控制
   - 测量单自动生成和智能派单
   - 安装单任务调度和质量验收
   - 状态同步机制，异常处理流程

3. **积分激励机制**
   - 多节点积分发放（创建/跟进/转化/成交）
   - 动态积分系数（产品/时间/门店）
   - 实时积分计算和到账提醒
   - **在途积分管理**：订单确认后积分进入在途状态，保护公司权益同时激励设计师
   - 积分商城兑换和管理

4. **服务商管理**
   - 技能认证和标签管理
   - 智能匹配算法（技能+地理+负荷）
   - 绩效评估体系（准时率/质量/投诉）
   - 收入结算和提现管理

5. **罗莱好伙伴管理**
   - **伙伴信息管理**：圣都设计师、圣都导购档案
   - **积分激励体系**：基于订单金额的积分分配和发放机制
   - **专属移动端**：好伙伴专属小程序界面，支持积分查询和礼品兑换
   - **订单协同**：7天延迟确认机制，确保订单信息准确无误
   - **数据分析**：伙伴业绩分析，转化率统计，贡献度评估
   
   **角色定义与职责**：
   - **设计师角色**：
     - 负责初步方案设计和专业建议
     - 参与草签阶段的方案讨论和优化
     - 提供专业的设计咨询和产品搭配建议
     - 协助驻店销售完善客户需求分析
   - **导购角色**：
     - 负责产品推荐和客户沟通
     - 协助草签阶段的产品选择和配置
     - 提供产品知识支持和价格咨询
     - 参与客户关系维护和跟进服务
   
   **协作机制**：
   - 每个订单可指定最多2个罗莱好伙伴（设计师+导购）
   - 在草签状态下提供专业支持，但不具备订单操作权限
   - 通过积分激励机制获得相应回报
   - 与驻店销售形成协同服务团队

6. **客户自助服务**
   - 小程序端订单状态实时查看
   - 设计方案确认与反馈
   - 安装进度跟踪与验收确认
   - 客户满意度评价与反馈收集
   - 电子合同签署与查看

7. **财务对账与回款管理**
   - 销售数据与财务数据自动对账
   - 多节点回款状态跟踪（定金/进度款/尾款）
   - 收入确认与会计凭证生成
   - 发票管理与开具流程
   - 回款预警与催收提醒

8. **商品与库存管理**
   - 多品类商品管理（窗帘/墙布/墙咔）
   - 商品规格与价格体系维护
   - 库存实时监控与预警
   - 供应商管理与采购协同
   - 商品组合与套餐配置

9. **数据报表与分析**
   - L2C销售漏斗分析（线索→成交→回款）
   - 销售业绩多维度分析（人员/门店/时间/品类）
   - 服务效率分析（测量/安装时效）
   - 客户满意度与投诉分析
   - 积分激励效果分析
   - 财务指标分析（收入/成本/利润）

## 🔄 线索转化与订单状态业务规则

### 线索管理业务规则

#### 1. 线索状态定义
**线索状态流转**：待分配 → 待跟踪 → 跟踪中 → 草签 → 已关闭

#### 2. "草签"动作详细说明
**动作性质**：
- "草签"是一个业务动作，不是订单状态
- 是线索转化为订单的关键操作节点
- 是对线索跟踪的延续，承接线索管理的后续流程
- 完成草签动作后，订单状态进入"等待推单"

**线索转化完整流程**：
1. **线索转化路径**（B2B客户）：
   - 线索管理页面 → 点击"预报价"按钮 → 上传Excel文件 → 线索获得"quoted"标签
   - 点击"转化"按钮 → 执行"草签"动作 → 订单状态进入"预算中"
   - 进行成本预算和方案设计 → 上传《预算报价单》→ 订单状态转为"等待推单"
   - 预报价信息自动继承到新创建的订单中

2. **直接草签路径**（零售客户）：
   - 零售客户可直接执行"草签"动作创建订单
   - 无需经过线索管理和预报价环节
   - 草签完成后订单进入"预算中"状态
   - 完成预算设计并上传《预算报价单》后进入"等待推单"状态

3. **合作伙伴路径**：
   - 外部合作伙伴（导购 → 设计师）→ 业务

**草签动作必须信息**：
- 客户基本信息（从线索继承或新建时录入）
- 项目基本信息（地址、面积、需求描述等）
- **预计测量时间**：不需要锁定，可根据实际情况灵活调整
- 初步预算范围

**《预算报价单》要求**：
- **文档性质**：正式的预算报价文档，区别于线索阶段的Excel预报价文件
- **生成时机**：订单进入"预算中"状态后，完成成本预算和方案设计
- **上传要求**：支持PDF、Word、Excel等格式，文件大小不超过10MB
- **必含内容**：详细商品清单、准确价格、施工方案、交付时间
- **状态触发**：上传完成后，订单状态自动从"预算中"转为"等待推单"
- **版本管理**：支持多次上传，保留历史版本记录

**草签操作权限**：
- **主要操作者**：驻店销售人员
- **协助角色**：外部合作伙伴（可指定2个角色）
  - **设计师**：负责初步方案设计和专业建议
  - **导购**：负责产品推荐和客户沟通
- 驻店销售具有草签动作的完整执行权限
- 外部合作伙伴在指定角色范围内提供专业支持

#### 3. 推单业务流程说明

**推单定义**：
推单是指圣都Home业务系统中发出采购申请后，作为供货方（罗莱）需要完成的推单确认操作。这是连接圣都内部采购流程与罗莱供货流程的关键节点。

**推单操作详解**：
- **操作角色**：远程销售负责上传，订单客服负责确认
- **操作时机**：订单状态为"待推单"时执行
- **操作流程**：
  1. 远程销售上传圣都Home系统采购申请截图
  2. 订单客服审核采购申请截图，确认采购需求
  3. 确认完成后订单状态变为"待下单"
- **截图要求**：必须包含完整的采购申请信息
  - 采购申请单号
  - 产品明细清单（名称、规格、数量）
  - 采购金额
  - 申请部门和申请人信息
  - 申请时间戳

**推单业务逻辑**：
1. **圣都内部流程**：圣都员工在Home系统提交采购申请
2. **推单生成**：系统自动生成推单，通知供货方（罗莱）
3. **推单确认**：订单客服审核远程销售上传的采购申请截图，确认采购需求后完成推单确认
4. **状态变更**：推单确认完成后，订单状态从"待推单"变为"待下单"
5. **后续流程**：进入生产下单环节，由订单客服处理

**推单失败处理**：
- **截图不符合要求**：可重新上传符合规范的截图
- **推单信息不匹配**：订单客服联系圣都相关人员核实采购申请信息
- **采购需求不明确**：订单客服要求远程销售补充更完整的采购申请信息
- **内部确认流程**：如订单客服对采购需求有疑问，可向上级申请复核
- **系统异常**：记录异常原因，申请技术支持处理

**推单时效要求**：
- **上传时效**：远程销售需在24小时内完成采购申请截图上传
- **确认时效**：订单客服需在24小时内完成采购需求确认
- **总处理时效**：48小时内完成整个推单确认流程
- **超时提醒**：48小时系统自动提醒，72小时升级到上级
- **紧急处理**：特殊情况下可申请加急处理

#### 4. "quoted"标签有效期规则
**有效期设置**：
- 预报价有效期：90天
- 系统每日凌晨2点自动刷新状态

**跟踪提醒机制**：
- 第7天：首次跟踪提醒
- 第30天：重点跟踪提醒
- 第60天：紧急跟踪提醒
- 第90天：即将过期提醒

**状态更新规则**：
- 超过10天未转化：自动添加"需维护"标签
- 超过90天未转化：保持"quoted"状态，但添加"需维护"标签
- 已报价线索不自动更新价格，保持历史报价准确性

**数据保留规则**：
- 预报价记录保存1年
- 即使线索取消，预报价信息也需保留供查询

### 销售单状态流转规则

#### 1. 销售单状态定义（27个状态）

**线索阶段状态（5个）**
1. **待分配**：线索创建后等待分配给销售人员
2. **待跟踪**：已分配但尚未开始跟踪
3. **跟踪中**：正在跟踪客户
4. **草签**：客户初步同意，准备进入测量阶段
5. **已失效**：手动关闭的线索

**订单阶段状态（15个）**
6. **待测量**：等待安排测量师上门测量
7. **测量中-待分配**：测量需求单已生成，等待派单员分配
8. **测量中-分配中**：派单员正在分配测量师
9. **测量中-待上门**：测量师已接单，等待上门测量
10. **测量中-待确认**：测量完成，等待销售确认测量单
11. **方案待确认**：测量单已确认，等待客户确认方案
12. **待推单**：方案已确认，等待推单处理
13. **待下单**：推单完成，等待下单
14. **生产中**：订单已下单，正在生产
15. **备货完成**：所有生产单号备货完成
16. **安装中-待分配**：安排安装，等待派单员分配安装师
17. **安装中-分配中**：派单员正在分配安装师
18. **安装中-待上门**：安装师已接单，等待上门安装
19. **安装中-待确认**：安装完成，等待销售确认
20. **待对账**：安装确认完成，等待财务对账

**财务阶段状态（2个）**
21. **待回款**：财务已开具销售发票并填写发票单号，等待客户回款（注：根据业务流程，需要先开具发票才能进入回款环节）
22. **已完成**：回款确认完成，订单结束

**异常状态（5个）**
23. **已取消**：订单被取消
24. **暂停**：订单暂停处理
25. **异常**：订单出现异常情况

#### 2. 状态流转条件
**销售单创建流程**：
线索录入 → 待分配 → 待跟踪 → 跟踪中 → 草签 → 待测量

**正常流转路径**：
待分配 → 待跟踪 → 跟踪中 → 草签 → 待测量 → 测量中-待分配 → 测量中-分配中 → 测量中-待上门 → 测量中-待确认 → 方案待确认 → 待推单 → 待下单 → 生产中 → 备货完成 → 安装中-待分配 → 安装中-分配中 → 安装中-待上门 → 安装中-待确认 → 待对账 → 待回款 → 已完成

**异常处理机制**：
- 任何状态都可以转为"已取消"、"暂停"、"异常"
- 支持状态回退：如测量数据有误可从"待确认"回退到"待测量"
- 异常状态可恢复到正常流程的任意节点

#### 3. 关键检查点与确认时机
**客户确认节点**：
- 预算中 → 等待推单：上传《预算报价单》后自动流转
- 待确认 → 待推单：需要客户确认最终方案和价格
- 待对账 → 待回款：需要客户确认安装质量和费用

**内部操作节点**：
- 等待推单 → 待测量：派单员点击"已接单"按钮
- 待测量 → 待出方案：测量数据上传和确认流程
  - **未来版本**：师傅上传测量数据 → 派单员点击"确认"按钮
  - **当前开发阶段**：派单员直接填写测量数据后转化
- 待出方案：设计师完成方案设计
- 待下单：销售确认订单信息无误
- 待安装：确认产品到货和安装条件

**供应商协调节点**：
- 待下单 → 待生产：向供应商下单
- 待生产 → 待安装：确认生产完成和发货

### 测量数据管理规则

#### 1. 测量数据上传流程
**未来完整版本流程**：
1. **师傅端操作**：
   - 师傅完成现场测量后，通过移动端上传测量数据
   - 支持上传测量图纸、尺寸数据、现场照片
   - 系统自动记录上传时间和师傅信息

2. **派单员确认流程**：
   - 派单员收到测量数据上传通知
   - 审核测量数据的完整性和准确性
   - 点击"确认"按钮后，订单状态从"待测量"转为"待出方案"

**当前开发阶段临时方案**：
- 派单员直接在系统中填写测量数据
- 填写完成后，订单状态自动从"待测量"转为"待出方案"
- 保留数据结构，便于未来升级到完整版本

#### 2. 测量数据要求
**必填数据项**：
- 房间尺寸（长、宽、高）
- 门窗位置和尺寸
- 特殊结构说明
- 现场环境照片（当前阶段可选）

**数据格式规范**：
- 尺寸单位：毫米(mm)
- 照片格式：JPG/PNG，单张不超过5MB
- 图纸格式：PDF/JPG/PNG

#### 3. 权限和操作规则
**操作权限**：
- **师傅**：上传测量数据（未来版本）
- **派单员**：确认测量数据、填写测量数据（当前版本）
- **设计师**：查看测量数据，用于方案设计

**状态流转触发**：
- 测量数据确认完成 → 自动流转到"待出方案"
- 数据不完整或有误 → 保持"待测量"状态，发送补充通知

### 系统自动化规则

#### 1. 定时任务设置
**每日凌晨2点执行**：
- 刷新所有线索的"quoted"标签状态
- 检查并添加"需维护"标签（超过10天的线索）
- 生成状态变更日志

#### 2. 自动提醒机制
**销售跟踪提醒**：
- 基于"quoted"标签的分级提醒
- 系统消息、邮件、短信多渠道提醒

**订单状态提醒**：
- 状态停留时间超时提醒
- 关键节点确认提醒
- 异常状态处理提醒

#### 3. 数据管理规则
**历史数据保留**：
- 预报价记录：1年
- 订单状态变更日志：永久保留
- 线索跟踪记录：永久保留

**数据同步规则**：
- 线索转化时，预报价信息自动继承
- 状态变更实时同步到相关系统
- 财务数据与订单状态联动更新
