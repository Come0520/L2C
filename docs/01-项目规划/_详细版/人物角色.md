# 人物角色定义

## 概述
本文档定义了L2C系统中所有参与业务流程的人物角色，明确各角色的职责范围、操作权限和业务流程中的作用。

---

## 角色覆盖说明

- Web端登录角色：ADMIN、SUPER_ADMIN、SALES_MANAGER、STORE_SALES、REMOTE_SALES、LEAD_VIEWER、系统管理员、FINANCE、订单客服（ORDER_SERVICE）、派单员、MANAGEMENT_APPROVER
- 不登录Web端：USER（客户）仅通过小程序端（Taro）访问系统，不作为Web登录角色；客户相关功能通过小程序完成订单状态查看、方案确认、物流追踪、评价与积分等。
- 小程序端登录角色（不登录 Web）：测量师、安装师、罗莱好伙伴设计师（PARTNER_DESIGNER）、导购（PARTNER_GUIDE）
- 审批角色说明：BUSINESS_APPROVER、FINANCIAL_APPROVER 为审批角色标签（权限维度），非真实身份角色，不单独作为登录主体，仅在审批流程中用于权限路由与策略控制
- 待补充角色（未在当前正文中详细定义）：
  - 采购专员（PURCHASER）：供应商下单、到货验收、采购结算（关联模块：供应商管理、库存管理、财务管理）
  - 库管（WAREHOUSE_KEEPER）：入库/出库、盘点、移库、损耗管理（关联模块：库存管理、物流管理）
  - 物流专员（LOGISTICS_SPECIALIST）：承运商管理、签收异常处理、在途监控（关联模块：物流管理）
  - 售后客服（AFTER_SALES_SERVICE）：退换货、返修、投诉处理（关联模块：订单管理、安装管理、通知系统）
  - 质检（QUALITY_ASSURANCE）：来料/生产/安装质检、返工判定（关联模块：项目交付、状态流转规则）
  - 积分管理员（POINTS_ADMIN）：积分策略配置、清分与审计（关联模块：积分系统）
  - 商城运营（MALL_OPERATOR）：上架、价格与促销、兑换规则、活动管理（关联模块：积分商城）
  - 法务合规（LEGAL_COMPLIANCE）：合同条款审查、风险条款监督（关联模块：合同管理、审批流程）
  - 风控/信用审核（RISK_CONTROLLER）：客户授信与订单信用审核（关联模块：财务管理、审批流程）
  - 数据分析师（DATA_ANALYST）：指标建模、报表构建、数据治理（关联模块：报表分析、数据分析）
  - 产品管理员（PRODUCT_MANAGER）：规格库与内容管理（关联模块：产品管理、文件与分享）
  - 供应链计划（S&OP_PLANNER）：产能/库存/交期计划（关联模块：库存管理、物流管理、订单管理）


## 角色分类

### 1. 销售类角色

#### 1.1 驻店销售
**角色描述**：在实体门店工作的销售人员，直接面对到店客户
**主要职责**：
- 接待到店客户，提供产品咨询和现场展示
- 线索跟踪：点击"跟踪"按钮，记录跟踪情况
- 标签管理：为线索添加产品类、装修阶段类、意愿强度标签
- 报价流程：点击"报价"按钮，完成报价单填写，自动计算预算
- 到店确认：点击"到店"按钮，标记客户到店状态
- 转化操作：点击"转化"按钮，将线索转为草签状态
- 草签操作：点击"草签"按钮，完成报价单填写，在授权折扣范围内确定成交金额
- 测量安排：点击"去测量"按钮，生成测量需求单，在报价单基础上添加相关备注
- 测量确认：对测量单数据进行确认，可选择"驳回"或"确认"
- 方案确认：上传客户确认的图片，推进订单流程
- 安装安排：点击"安排安装"按钮，填写安装备注（如进门密码等）
- 安装确认：上传安装后的图片，完成安装验收
- 线索关闭：点击"关闭"按钮，关闭线索（可重新激活）

**业务权限**：
- 线索跟踪：状态更新、跟踪记录、标签添加
- 接收分配线索：接收上级分配的线索进行跟踪
- 草签动作执行权限（需渠道负责人授权最低折扣额）
- 测量需求单生成和确认权限
- 安装安排和确认权限

**角色描述**：通过电话、网络等远程方式进行销售的人员
**主要职责**：
- 线索跟踪：点击"跟踪"按钮，记录跟踪情况
- 电话回访和客户跟进，维护客户关系
- 标签管理：为线索添加产品类、装修阶段类、意愿强度标签
- 线上客户咨询处理和远程产品介绍
- 报价流程：点击"报价"按钮，完成报价单填写，自动计算预算
- 转化操作：点击"转化"按钮，将线索转为草签状态
- 草签操作：点击"草签"按钮，完成报价单填写，在授权折扣范围内确定成交金额
- 测量确认：对测量单数据进行确认，可选择"驳回"或"确认"
- 方案确认：上传客户确认的图片，推进订单流程
- 推单处理：上传圣都Home系统采购申请截图，由订单客服确认采购需求后，将订单推进到待下单状态（仅远程销售可操作）
- 安装安排：点击"安排安装"按钮，填写安装备注（如进门密码等）
- 安装确认：上传安装后的图片，完成安装验收
- 线索关闭：点击"关闭"按钮，关闭线索（可重新激活）

**业务权限**：
- 线索跟踪：状态更新、跟踪记录、标签添加
- 客户信息查看和更新
- 电话记录和跟进记录
- 草签动作权限（需渠道负责人授权最低折扣额）
- 测量确认权限
- 推单上传权限：上传圣都Home系统采购申请截图权限（仅远程销售拥有）
- 安装安排和确认权限

- 线索跟踪：点击"跟踪"按钮，记录跟踪情况
- 标签管理：为线索添加产品类、装修阶段类、意愿强度标签

- 渠道合作伙伴管理和市场拓展活动
- 报价流程：点击"报价"按钮，完成报价单填写，自动计算预算
- 到店确认：点击"到店"按钮，标记客户到店状态
- 转化操作：点击"转化"按钮，将线索转为草签状态
- 草签操作：点击"草签"按钮，完成报价单填写，在授权折扣范围内确定成交金额
- 测量安排：点击"去测量"按钮，生成测量需求单，在报价单基础上添加相关备注
- 测量确认：对测量单数据进行确认，可选择"驳回"或"确认"
- 方案确认：上传客户确认的图片，推进订单流程
- 安装安排：点击"安排安装"按钮，填写安装备注（如进门密码等）
- 安装确认：上传安装后的图片，完成安装验收
- 线索关闭：点击"关闭"按钮，关闭线索（可重新激活）

**业务权限**：
- 线索创建：手动录入、Excel批量导入
- 线索分配：对导入的线索信息进行人员分配
- 线索跟踪：状态更新、跟踪记录、标签添加
- 高级线索管理权限和大客户信息管理
- 完整的草签动作权限
- 测量需求单生成和确认权限
- 安装安排和确认权限
- 接收分配线索：接收上级分配的线索进行跟踪处理



### 2. 领导类角色

#### 2.1 销售负责人
**角色描述**：销售团队的直接管理者
**主要职责**：
- 线索分配：对导入的线索信息进行人员分配
- 销售团队管理和考核，销售目标制定和监控
- 重要客户关系维护和销售流程优化
- 城市授权分配人员管理
- 到店确认：点击"到店"按钮，标记客户到店状态

**业务权限**：
- 线索分配：城市授权分配权限（最高层级分配权限，可分配给城市授权人员，可跨部门分配）
- 所有销售数据查看权限和销售人员业绩管理
- 价格审批权限和订单状态监控

#### 2.2 渠道负责人
**角色描述**：负责渠道合作和渠道管理的领导
**主要职责**：
- 渠道合作伙伴开发
- 渠道政策制定和执行
- 渠道业绩监控
- 渠道冲突处理

**业务权限**：
- 渠道数据分析权限
- 渠道政策配置
- 渠道订单特殊处理
- 渠道价格体系管理

#### 2.3 领导
**角色描述**：公司高级管理人员
**主要职责**：
- 整体业务战略制定
- 重大决策审批
- 业务流程监督
- 团队绩效评估

**业务权限**：
- 全系统数据查看权限
- 重大订单审批
- 系统配置管理
- 人员权限分配

#### 2.4 系统管理员
**角色描述**：负责系统管理、配置和维护的技术管理人员
**主要职责**：
- 系统配置管理：业务规则配置、系统参数设置
- 用户权限管理：角色权限分配、用户账号管理
- 数据备份和恢复：定期数据备份、灾难恢复
- 系统监控和维护：性能监控、故障排查
- 安全管理：访问控制、安全审计
- 技术支持：为其他角色提供技术支持和培训

**业务权限**：
- 系统配置权限：业务规则配置、系统参数设置
- 用户管理权限：用户账号创建、角色权限分配
- 数据管理权限：数据备份、数据恢复、数据导出
- 系统监控权限：性能监控、日志查看、故障诊断
- 安全管理权限：访问控制配置、安全策略设置
- 技术支持权限：系统维护、版本更新、技术培训

#### 2.5 领导查看角色 (LEAD_VIEWER)
**角色描述**：专门负责数据查看和分析的领导角色（只读权限，不具备任何业务操作和审批权限）
**主要职责**：
- 查看各类业务仪表板和数据报表
- 监控销售业绩和趋势变化
- 分析业务运营效率和流程质量
- 关注关键指标和异常数据预警
- 为管理决策提供数据支持
- **重要说明**：此角色仅具备数据查看权限，不能进行任何业务操作、审批决策或系统配置

**业务权限**：
- 仪表板查看：销售仪表板、运营仪表板、财务仪表板、自定义仪表板
- 销售数据查看：销售业绩、销售绩效、销售趋势、销售排名、转化率
- 业务数据查看：业务概览、流程效率、质量指标、客户分析
- 运营数据查看：运营指标、效率分析、质量监控、异常预警
- 报表查看：销售报表、财务报表、业务报表、自定义报表
- 实时数据监控：实时销售、实时运营、实时预警
- 数据导出：销售数据导出、业务数据导出、报表导出、图表导出
- 对比分析：同期对比、趋势分析、结构分析、预测分析

**典型使用场景**：
- 晨会数据汇报：每天早上查看前一天的业务数据汇总
- 销售趋势监控：实时关注销售业绩和趋势变化
- 运营效率分析：查看各部门运营效率和质量指标
- 异常数据预警：及时发现业务异常和风险点
- 月度总结分析：查看月度业务报表和分析总结
- 决策数据支持：为重要决策提供数据依据和支持

**权限限制说明**：
- ❌ 无法进行任何业务操作（如订单处理、客户管理等）
- ❌ 无法进行任何审批决策（如折扣审批、预算审批等）
- ❌ 无法进行系统配置和管理操作
- ❌ 无法导出敏感数据或进行数据修改
- ✅ 仅可查看授权范围内的业务数据和报表
- ✅ 可进行数据对比分析和趋势预测查看

### 3. 审批类角色

#### 3.1 业务审批人 (BUSINESS_APPROVER)
**角色描述**：负责业务相关审批事项的专业审批人员
**主要职责**：
- 处理业务相关的各类审批任务
- 审批客户特殊折扣申请
- 审批特殊报价和报价修改
- 为订单添加加急标签
- 提供审批意见和建议
- 在授权范围内进行委托审批和批量审批

**业务权限**：
- 业务审批：处理业务相关审批任务
- 客户审批：客户信用额度审批、客户特殊折扣审批
- 报价审批：特殊报价审批、报价折扣审批、报价修改审批
- 订单标签：为订单添加加急标签
- 审批管理：查看业务审批、添加审批意见、委托审批、批量审批

**具体审批示例**：
- **客户特殊折扣审批**：重要客户申请15%特殊折扣，需要评估客户价值、历史合作、未来潜力等因素
- **特殊报价审批**：项目报价超出标准价格体系，需要评估项目规模、竞争情况、利润空间等
- **加急订单标签**：客户要求加急生产安装，需要评估生产能力、资源调配、加急费用等

#### 3.2 财务审批人 (FINANCIAL_APPROVER)
**角色描述**：负责财务相关审批事项的专业审批人员
**主要职责**：
- 处理财务相关的各类审批任务
- 审批预算申请和预算调整
- 审批各类费用报销和支出
- 审批供应商付款和薪资发放
- 进行财务审计和风险控制
- 确保财务流程合规性

**业务权限**：
- 财务审批：处理财务相关审批任务
- 预算审批：预算申请审批、预算调整审批、预算超支审批
- 费用审批：费用报销审批、差旅费审批、采购费用审批
- 付款审批：供应商付款审批、薪资发放审批
- 财务审计：查看财务审批、进行财务审计、批量财务审批

**具体审批示例**：
- **预算申请审批**：市场部申请Q2推广预算80万元，需要评估预算合理性、ROI预期、历史效果等
- **费用报销审批**：销售总监申请2.8万元客户招待费报销，需要核实费用真实性、合规性、预算余额等
- **供应商付款审批**：采购部申请支付300万元原材料货款，需要核对合同条款、验收情况、付款条件等

#### 3.3 管理审批人 (MANAGEMENT_APPROVER)
**角色描述**：负责高级管理审批事项的企业高层管理人员
**主要职责**：
- 处理战略性、跨部门、高风险的重大审批事项
- 审批重大战略合同和投资决策
- 审批组织架构调整和人事任免
- 审批重要合作伙伴关系
- 处理危机事件和紧急事项
- 进行跨部门争议仲裁

**业务权限**：
- 管理审批：处理高级管理审批任务
- 战略审批：战略合同审批、投资决策审批、合作伙伴审批
- 组织审批：组织架构调整审批、人事任免审批、政策制度审批
- 风险审批：危机处理审批、合规风险审批、紧急事件审批
- 跨部门审批：跨部门预算审批、跨部门项目审批、部门争议仲裁
- 系统管理：查看全数据、查看管理数据、查看战略数据、管理工作流、查看审计日志

**具体审批示例**：**具体审批示例**（续）：
- **战略合同审批**：与重要合作伙伴签署5年期战略合作协议，涉及金额5000万元，需要评估合作方实力、市场前景、风险条款等
- **组织架构调整审批**：将销售部分为华东、华南、华北三个区域事业部，需要评估调整必要性、成本效益、人员安排等
- **危机处理审批**：主要供应商突然停产影响产品交付，需要快速决策应急方案、替代供应商、客户沟通等

**审批触发条件说明**：
- **业务审批触发**：系统自动检测销售草签中的折扣率，当超出预设阈值（如>15%）时自动触发业务审批人流
  - 触发示例：销售为客户申请18%折扣 → 系统拦截 → 生成业务审批任务 → 业务审批人评估决策
  
- **财务审批触发**：涉及预算、费用、付款的审批事项，根据金额阈值和类型自动路由到财务审批人
  - 触发示例：部门预算申请超预算20% → 系统识别为超预算 → 生成财务审批任务 → 财务审批人评估ROI
  
- **管理审批触发**：跨部门、战略性事项由系统识别并升级至管理审批人处理
  - 触发示例：5000万战略合作协议 → 系统识别为重大合同 → 生成管理审批任务 → 管理审批人战略评估

- 订单信息核实和处理
- 客户服务和问题解决
- 订单状态跟踪和更新
- 客户满意度维护

**业务权限**：
- 生产单号填写和管理权限
- 备货状态更新权限
- 物流单号填写权限
- **推单确认权限**：审核圣都Home系统采购申请截图，确认采购需求权限

### 5.2 编码规则

#### 线索编码规则
**编码格式**：前三位城市代码 + 6位数字
- 系统自动生成，确保唯一性
- 城市代码采用国家标准GB/T 2260
- 示例：SHA123456（上海线索）

#### 订单编码规则
**编码格式**：OR + 年月日 + 4位数字
- OR代表订单（Order）
- 年月日格式：YYYYMMDD
- 示例：OR202312150001（2023年12月15日第1个订单）

#### 客户编码规则
**编码格式**：C + 年月日 + 4位数字
- C代表客户（Customer）
- 年月日格式：YYYYMMDD
- 示例：C202312150001（2023年12月15日第1个客户）

#### 审批编码规则
**编码格式**：AP + 角色类型 + 年月日 + 4位数字
- AP代表审批（Approval）
- 角色类型：B（业务）、F（财务）、M（管理）
- 年月日格式：YYYYMMDD
- 示例：APB202312150001（2023年12月15日第1个业务审批）

- 中意愿：有购买意向，需进一步沟通
- 低意愿：了解阶段，暂无明确购买计划

#### 操作状态标签

**线索阶段状态（5个）**
- 待分配：线索创建后等待分配给销售人员
- 待跟踪：已分配但尚未开始跟踪
- 跟踪中：正在跟踪客户
- 草签：客户初步同意，准备进入测量阶段
- 已失效：手动关闭的线索

**客户相关标签**
- 已注册：客户完成注册
- 已绑定：客户绑定销售代表
- 已激活：客户开始活跃使用系统

**订单阶段状态（15个）**
- 待测量：等待安排测量师上门测量
- 测量中-待分配：测量需求单已生成，等待派单员分配
- 测量中-分配中：派单员正在分配测量师
- 测量中-待上门：测量师已接单，等待上门测量
- 测量中-待确认：测量完成，等待销售确认测量单
- 方案待确认：测量单已确认，等待客户确认方案
- 待推单：方案已确认，等待推单处理
- 待下单：推单完成，等待下单
- 生产中：订单已下单，正在生产
- 备货完成：所有生产单号备货完成
- 安装中-待分配：安排安装，等待派单员分配安装师
- 安装中-分配中：派单员正在分配安装师
- 安装中-待上门：安装师已接单，等待上门安装
- 安装中-待确认：安装完成，等待销售确认
- 待对账：安装确认完成，等待财务对账

**财务阶段状态（2个）**
- 待回款：财务已填写发票单号，等待客户回款
- 已完成：回款确认完成，订单结束

**异常状态（5个）**
- 已取消：订单被取消
- 暂停：订单暂停处理
- 异常：订单出现异常情况

**其他标签**
- 已到店：客户已到店面进行实地咨询
- 已报价：已完成报价单填写和预算计算
- 已发货：订单客服已填写物流单号，货物已发出

### 5.3 城市授权分配规则
- 每个城市设置授权分配人员
- 分配人员具备该城市线索分配权限
- 分配权限由销售负责人/业务担任

## 六、角色协作流程

### 6.1 线索阶段协作
- **线索创建**：业务/销售负责人 → 手动录入或Excel/CSV批量上传 → 状态自动变为"待分配" → 系统生成编号
- **线索分配**：销售负责人/业务 → 城市授权人员分配（优先分配给所在城市的门店销售，其次分配给远程销售支持）→ 状态自动变为"待跟踪"
- **线索跟踪**：驻店销售/远程销售 → 点击"跟踪"按钮 → 添加标签和跟踪记录 → 状态变为"跟踪中"
- **报价流程**：驻店销售/远程销售 → 点击"报价"按钮 → 完成报价单填写 → 状态变为"已报价"
- **到店确认**：驻店销售/业务/销售负责人 → 点击"到店"按钮 → 添加"已到店"标签
- **线索转化**：驻店销售/远程销售/业务 → 点击"转化"按钮 → 状态变为"草签"
- **草签确认**：驻店销售/远程销售 → 点击"草签"按钮 → 完成报价单填写和成交金额确定 → 状态变为"待测量"
- **测量安排**：驻店销售/远程销售 → 点击"去测量"按钮 → 生成测量需求单 → 状态变为"测量中-待分配"
- **线索失效**：驻店销售/远程销售/业务 → 点击"失效"按钮 → 状态变为"已失效"（客户无购买意向或长期无响应）
- **线索关闭**：驻店销售/远程销售/业务 → 点击"关闭"按钮 → 状态变为"已关闭"（可重新激活恢复到"跟踪中"）
- **待分配预警**：待分配状态停留超过24小时，系统自动预警，上报给领导和渠道负责人；超过48小时，当月扣5分；超过72小时，当月扣10分
- **待分配统计**：系统统计"待分配"状态的平均停留时间指标
- **待跟踪预警**：待跟踪状态停留超过72小时，系统自动提醒销售人员；超过96小时，预警给领导、销售负责人、渠道负责人；超过120小时，扣5分，线索自动退回待分配状态

### 6.2 角色协作流程

#### 6.2.1 销售流程协作
1. **线索获取**：驻店销售/远程销售 → 业务
2. **线索分配**：业务 → 驻店销售/远程销售/外部合作伙伴
3. **客户跟进**：销售角色 → 客户管理
4. **订单创建**：销售角色 → 订单客服
5. **订单处理**：订单客服 → 派单员 → 测量师 → 安装师
6. **售后服务**：客户 → 订单客服 → 相关服务角色
7. **客户参与**：客户 → 方案确认 → 服务评价
8. **伙伴协作**：外部合作伙伴（导购）→ 线索创建 → 外部合作伙伴（设计师）→ 方案设计

#### 6.2.2 审批流程协作
1. **业务审批**：销售角色 → 业务审批人
2. **财务审批**：销售角色 → 财务审批人
3. **管理审批**：销售角色 → 管理审批人
4. **数据查看**：领导查看角色 → 各类数据报表

#### 6.2.3 管理流程协作
1. **数据查看**：领导 → 销售负责人/渠道负责人/领导查看角色
2. **审批流程**：销售角色 → 业务审批人/财务审批人/管理审批人
3. **财务流程**：销售角色 → 财务 → 财务审批人
4. **系统配置**：系统管理员 → 领导 → 各角色

### 6.3 订单阶段协作
- **测量派单**：派单员 → 分配测量需求单给测量师 → 状态变为"测量中-分配中"
- **测量接单**：测量师 → 点击"接单"按钮 → 状态变为"测量中-待上门"
- **测量执行**：测量师 → 上门测量生成测量单 → 状态变为"测量中-待确认"
- **测量确认**：驻店销售/远程销售 → 确认测量单（可驳回重新分配）→ 状态变为"方案待确认"
- **方案确认**：驻店销售/远程销售 → 上传客户确认图片 → 状态变为"待推单"
- **推单处理**：远程销售 → 上传圣都Home系统采购申请截图 → 订单客服确认 → 状态变为"待下单"（说明：推单是指圣都Home系统中发出采购申请，作为供货方需由订单客服确认采购需求后，订单才能进入待下单状态）
- **生产管理**：订单客服 → 填写生产单号 → 状态变为"生产中"
- **备货完成**：订单客服 → 所有生产单号备货完成 → 状态变为"备货完成"
- **安装安排**：驻店销售/远程销售 → 点击"安排安装"按钮 → 状态变为"安装中-待分配"
- **物流管理**：订单客服 → 填写物流单号 → 添加"已发货"标签
- **安装派单**：派单员 → 派单给安装师 → 状态变为"安装中-分配中"
- **安装接单**：安装师 → 点击"接单"按钮 → 状态变为"安装中-待上门"
- **安装执行**：安装师 → 上传安装照片 → 状态变为"安装中-待确认"
- **安装确认**：驻店销售/远程销售 → 上传安装后图片 → 状态变为"待对账"

### 6.4 渠道协作流程
- **外部合作伙伴引流流程**：外部合作伙伴（导购）→ 线索录入 → 外部合作伙伴（设计师）→ 方案设计 → 客户确认 → 订单处理
- **客户参与流程**：客户 → 方案查看 → 方案确认 → 安装验收 → 服务评价
- **积分激励流程**：线索转化成功 → 系统自动计算积分 → 积分发放到个人账户 → 合作伙伴查看积分 → 申请积分兑换

### 6.4 财务阶段协作
- **发票管理**：财务 → 开具销售发票并填写发票单号 → 状态变为"待回款"（注：根据业务流程，需要先开具发票才能进入回款环节）
- **回款确认**：财务 → 上传回款图片并勾选对应单号 → 状态变为"已完成"
- **预算审核**：销售负责人 → 财务人员确认
- **合同签署**：销售人员 → 财务人员备案
- **成本核算**：财务人员 → 利润分析报告

## 角色流转关系

### 业务流程中的角色协作
1. **线索阶段**：驻店销售/业务/销售负责人/外部合作伙伴（导购）（创建） → 销售负责人/业务/驻店销售（分配） → 驻店销售/远程销售/外部合作伙伴（跟踪、报价、转化、草签）
2. **测量阶段**：驻店销售/远程销售（测量安排） → 派单员（分配） → 测量师（接单、上门、测量） → 驻店销售/远程销售（确认）
3. **方案阶段**：外部合作伙伴（设计师）→ 方案设计 → 客户（方案确认）
4. **生产阶段**：远程销售（推单） → 订单客服（生产单号、备货管理、物流）
5. **安装阶段**：驻店销售/远程销售（安装安排） → 派单员（分配） → 安装师（接单、上门、安装） → 驻店销售/远程销售（确认）
6. **财务阶段**：财务（发票单号、回款确认） → 订单完成
7. **评价阶段**：客户（服务评价） → 系统记录

### 权限升级路径
1. **基础销售路径**：驻店销售 → 远程销售 → 业务 → 销售负责人
2. **管理路径**：销售负责人 → 渠道负责人 → 领导
3. **专业路径**：测量员/安装员 → 调度员 → 渠道负责人
4. **财务路径**：订单客服 → 财务 → 领导
5. **合作伙伴路径**：罗莱好伙伴（圣都导购 → 圣都设计师）→ 业务

### 渠道角色特殊说明
1. **外部合作伙伴**：外部引流合作伙伴类型角色，非罗莱体系正式员工，包含以下子角色：
- **导购**：专注于渠道的销售业务，负责线索创建、跟踪和客户管理，通过成功引流获得积分奖励
- **设计师**：专注于渠道的设计服务，负责设计方案制作和图纸管理，通过成功引流获得积分奖励
- 所有外部合作伙伴角色的权限都限制在对应渠道范围内
   - 通过积分激励系统回馈引流贡献，可查看个人积分余额和申请兑换
