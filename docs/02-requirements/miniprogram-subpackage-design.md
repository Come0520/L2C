## 一、设计总览

小程序定位为 **外部服务窗口**，服务三类外部用户（客户、师傅、销售移动端），以及一个共享模块（云展厅）。

| 分包              | 服务角色  | 页面数  | 核心定位                                                 |
| :---------------- | :-------- | :-----: | :------------------------------------------------------- |
| `packageCustomer` | 客户      |    8    | 客户全生命周期（报价→签字→跟踪→验收→评价→报修→转介绍）   |
| `packageWorker`   | 师傅/工人 |   10    | 工人全生命周期（入驻→接单议价→日历→测量→安装→结算→判责） |
| `packageSales`    | 销售      |    5    | 移动碎片化轻量工具（线索操作→跟进→分享→量尺审查）        |
| `packageShowroom` | 共享      |    5    | 品牌橱窗（商城式卡片浏览→收藏→分享）                     |
| **主包**          | 全部角色  |   ~8    | 认证流程 + TabBar 页（工作台/线索/展厅/任务/我的）       |
| **合计**          |           | **~36** |                                                          |

### 架构目录结构

```
│   ├── tasks/                  TabBar 3：任务

│
├── packageCustomer/          ── 分包 A: 客户 (<2MB) ──

```

## 二、packageCustomer（客户全生命周期）

### 定位

客户通过小程序完成与品牌的所有交互。小程序是客户的**唯一数字触点**，也是获客引擎的核心。

### 客户旅程时间线

```
销售/老客户 分享展厅链接（微信分享卡）
   ↓
客户打开小程序 → 浏览展厅（主包 showroom）
   ↓
感兴趣 → 收到销售发来的报价单链接
   ↓
📋 查看报价单（quote-view）→ ✍️ 电子签字确认（quote-sign）
   ↓
下单 → 🚚 跟踪订单/工人进度（order-track）
   ↓
安装完成 → ✅ 验收确认（acceptance）
   ↓
使用一段时间 → ⭐ 评价（review）或 🔧 报修（after-sales）
   ↓
满意 → 🎁 推荐给朋友（refer-share）→ 朋友看到品牌落地页（refer-landing）
```

### 页面清单

| 序号 | 页面            | 核心功能                                                           | 优先级 |
| :--: | :-------------- | :----------------------------------------------------------------- | :----: |
|  1   | `quote-view`    | 只读查看销售发来的报价明细（商品列表、金额、备注）                 | 🔴 高  |
|  2   | `quote-sign`    | Canvas 手写签名 + 确认提交                                         | 🔴 高  |
|  3   | `order-track`   | 订单进度时间线（已派单→工人签到→量尺中→安装中→完工）               | 🟡 中  |
|  4   | `acceptance`    | 安装验收：拍照确认 + 签字                                          | 🟡 中  |
|  5   | `review`        | 对服务进行打星（1-5）+ 文字评价                                    | 🟢 低  |
|  6   | `after-sales`   | 报修申请（选报修类型 + 拍照 + 文字描述）                           | 🟡 中  |
|  7   | `refer-share`   | 老客户转介绍：生成带**租户品牌 logo** 的分享卡片/海报              | 🔴 高  |
|  8   | `refer-landing` | 被推荐新客户的品牌落地页：租户 logo + slogan + 精选案例 + 预约入口 | 🔴 高  |

### 关键设计决策

- `refer-landing` 为**独立页面**（方案 B），专为被推荐的新客户提供品牌首印象，不复用通用 landing 页
- 转介绍分享卡片体现**租户品牌标识**，增强营销价值

## 三、packageWorker（工人全生命周期）

### 定位

师傅是**线下执行者**，通过小程序完成从入驻到结算的全部流程。不接触 Web 后台。

### 师傅旅程时间线

```
入驻平台 → 📖 学习操作规范 + 📝 签署电子合同 + ✅ 答题考核上岗
   ↓
被推送订单（测量单或安装单）→ 查看详情/位置/时间/价格
   → 接单 / 填期望价还价 / 拒绝
   ↓
等待上门 → 📅 日历提醒（测量+安装两种任务）
   → 接收销售变更通知（时间/地址变更）→ 点击确认
   ↓
上门测量 → 📍 打卡 → 📐 填数据到报价单对应位置
   → 📸 图片 / 🎤 语音 / 🎥 视频回传 → 支持多方案测量
   ↓
（可能）销售对测量数据有异议 → 💬 沟通 → 安排再次上门
   ↓
安装 → 📍 打卡 → 📸 拍照上传
   ↓
💰 结算 → 查看已结/待结/预估收入（参考滴滴司机端/饿了么骑手端）
   ↓
（如有）售后判责 → 查看责任判定 → ✍️ 签字确认 → 金额计入扣款
```

### 页面清单

| 序号 | 页面              | 核心功能                                                                   | 优先级 |
| :--: | :---------------- | :------------------------------------------------------------------------- | :----: |
|  1   | `onboarding`      | 操作规范学习 + 电子合同签署 + 答题考核上岗（**支持开关：必考/选配**）      | 🟡 中  |
|  2   | `order-bid`       | 接收推单（测量单或安装单）→ 查看详情/价格 → **接单 / 填期望价还价 / 拒绝** | 🔴 高  |
|  3   | `schedule`        | 日历视图，展示**测量 + 安装**两种待办；接收变更推送 → 点击确认             | 🔴 高  |
|  4   | `task-detail`     | 单个任务完整信息（地址/联系人/报价单关联/沟通记录）                        | 🔴 高  |
|  5   | `measure`         | 测量打卡 + 数据填入报价单对应字段 + 图片/语音/视频 + **多方案支持**        | 🔴 高  |
|  6   | `measure-dispute` | 销售对测量数据有异议时的沟通记录 + 再次上门安排                            | 🟡 中  |
|  7   | `install-upload`  | 安装打卡 + 现场拍照上传                                                    | 🔴 高  |
|  8   | `settlement`      | 收入面板：已结算 / 待结算 / 预估收入（参考滴滴/饿了么）                    | 🟡 中  |
|  9   | `liability`       | 售后判责通知 → 查看详情 → 签字确认 → 金额自动计入扣款                      | 🟡 中  |
|  10  | `engineer`        | 工程师专属工作台入口（已有）                                               | 🟢 低  |

### 关键设计决策

- **议价模式选 A**：师傅填写期望价格提交，**派单员在 Web 端审批**（不是销售在小程序审批）
- **日历包含两种任务类型**：测量 + 安装，都支持议价
- **答题考核可配置**：租户可在后台设置是「必须通过才上岗」还是「选配学习」

## 四、packageSales（销售轻量移动端）

### 定位

销售的**重度操作**（制作报价单、CRM 管理、数据报表）在 Web 端完成。小程序是**碎片化移动工具**，只做"在外面跑业务时的快速操作"。

### 销售使用场景

```
外出展业 → 📲 转发展厅链接给客户
   ↓
有意向 → 📋 快速查看线索详情
   ↓
需要跟进 → ✍️ 写一条跟进记录 + 设置下次跟踪日期
   ↓
线索状态变化 → 修改阶段 / 放弃退回公海 / 从公海认领
   ↓
师傅已量尺 → 查看量尺数据 → 确认 / 标注异议
```

### 页面清单

| 序号 | 页面              | 核心功能                                                         | 优先级 |
| :--: | :---------------- | :--------------------------------------------------------------- | :----: |
|  1   | `lead-detail`     | 线索详情只读查看（底部集成操作按钮）                             | 🔴 高  |
|  2   | `lead-actions`    | 线索状态操作：**修改阶段**、**放弃（退回公海）**、**从公海认领** | 🔴 高  |
|  3   | `quick-follow-up` | 快速跟进记录 + **设置下次跟踪日期** + 可选图片附件               | 🔴 高  |
|  4   | `share-hub`       | 展厅分享中心：选素材/案例 → 生成微信分享卡                       | 🟡 中  |
|  5   | `measure-review`  | 查看师傅量尺数据 → 确认 / 标注异议                               | 🟡 中  |

### 关键设计决策

- **不在小程序做**：线索创建、报价单制作、完整 CRM 管理、数据报表 → 全部留给 Web 端
- `lead-detail` 和 `lead-actions` 在开发时**可能合并为一个页面**（详情页底部直接挂操作按钮）
- 跟进记录增加了**下次跟踪日期**字段，支持微信消息推送提醒

## 五、packageShowroom（云展厅共享）

### 定位

品牌橱窗，**商城式卡片布局**，纯展示 + 收藏，不做交易转化。Sales 和 Customer 共享使用。

### 页面清单

| 序号 | 页面             | 核心功能                                         | 优先级 |
| :--: | :--------------- | :----------------------------------------------- | :----: |
|  1   | `product-detail` | 商品详情大图（规格、风格标签、案例关联）         | 🔴 高  |
|  2   | `case-detail`    | 施工案例详情（前后对比照、房型、业主评价）       | 🔴 高  |
|  3   | `article-detail` | 知识文章（风格指南、保养技巧等软文内容）         | 🟡 中  |
|  4   | `favorites`      | 我的收藏列表（商品 + 案例 + 文章的统一收藏面板） | 🟡 中  |
|  5   | `capsule`        | 分享胶囊（生成精美品牌分享图卡片 → 转发微信）    | 🔴 高  |

### 关键设计决策

- 展厅首页（`pages/showroom/index`）在主包，采用**商城式卡片瀑布流布局**
- **不设购买/下单按钮**，纯展示 + 收藏，转化由销售驱动
- 收藏功能跨类型统一管理（商品/案例/文章可以共存于一个收藏列表）
