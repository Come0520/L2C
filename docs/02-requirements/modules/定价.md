# Pricing (定价) 模块需求设计文档

## 1. 模块定位
Pricing（定价引擎）模块为核心业务流程（特别是**报价协同**）提供智能定价指导。该模块基于数据分析（历史订单、近期报价、分类均价），动态提供安全且合理的建议价格及毛利预估。

## 2. 核心功能

### 2.1 定价参考 (`getPricingHintsAction`)
- **基础护城河 (Floor Price 防线)**：建议价 (`suggestedPrice`) 在任何计算规则下，绝对不允许低于此基础底价。这是防止严重亏单和业务故障的核心策略。
- **数据化建议**：综合提取并返回四个维度数据：
    1. 基础预估（底层拉齐：成本价 cost、底价 floorPrice、指导价 guidancePrice）。
    2. 历史成交概览（近 6 个月最高、最低及平均成交价）。
    3. 最近报价热度（近期发出的报价及其平均单价）。
    4. 类目竞价（同类别产品的平均市场指引价）。
- **毛利预估运算**：基于成本实时计算实际毛利 (`actual`)、预估毛利 (`estimated`) 和指导毛利 (`guidance`)。当毛利低于预警线 (`LOW_MARGIN_THRESHOLD` 20%) 时，触发前端视觉告警。
- **趋势分析**：按月返回近 6 个月的动态成交价格趋势。

## 3. 业务协同与隔离

### 3.1 报价协同 (Quote Synergy)
- 直接向报价（Quotes）模块和订单模块提供智能辅助数据。在前端报价表单（Create/Edit）输入或调整产品 SKU 时，调用该组件进行指导。

### 3.2 数据隔离与权限
- 所有数据拉取强制带入 `tenantId`，确保只能分析本租户的交易行为库。
- 权限隔离（RBAC机制）：默认仅允许包含 `SALES` 或高阶职位的组调用该建议，防备核心成本泄露。

## 4. 技术与性能优化
- **双层缓存架构**：
    - `unstable_cache` 负责宏观统计数据（如历史趋势）的缓存，减少数据库的大量 `COUNT` 和 `AVG` 聚合压力。
    - `React cache` 控制单次请求上下文内的防抖和调用去重。
- **审计与追踪**：每次获取定价建议（尤其是低于安全毛利红线的操作）和响应时长均会被日志追踪模块记录，为日后优化及漏斗溯源提供依据。
