# 搜索模块 (Search)

## 模块定位
提供系统级别的全局跨表数据搜索功能，并在底层集成 RBAC 权限过滤与关键词高亮功能，同时利用 Redis 处理搜索历史缓存。

## 当前支持的搜索范围
全局搜索接口支持根据用户当前权限及传入的 `scope` 参数，对以下 8 大核心关联模块/实体进行搜索：

1. **客户 (Customers)**: 根据客户名称、电话匹配。
2. **线索 (Leads)**: 根据关联客户的名称、电话匹配。
3. **订单 (Orders)**: 根据订单号匹配。
4. **报价 (Quotes)**: 根据报价单号匹配。
5. **产品 (Products)**: 根据产品名称、SKU 匹配。
6. **售后工单 (Tickets)**: 根据工单号匹配。
7. **渠道 (Channels)**: 根据渠道名称匹配。
8. **财务账单 (Finance)**: 根据对账单号匹配。

## 实现方式
- **权限安全**: `performDbSearch` 内部使用 `hasPerm` 辅助函数，严格校验用户本身绑定的 `permissions` 列表，如果用户不包含对应模块的可读权限，则直接跳过该模块的数据库检索。
- **并发优化**: 底层利用 `Promise.all` 实现多表的并发查询，显著降低长链路查询耗时。
- **高频词高亮**: 查询结果将被送入 `highlightText` 将匹配到的关键词部分使用 `<mark>` 标签包裹，以便前端直接渲染高亮。
- **历史记录缓存**: 使用 Redis (`search:history:tenantId:userId`) 结构缓存最近 10 条有效搜索记录，并在用户以空字符串搜索时下发作为搜索历史建议。
