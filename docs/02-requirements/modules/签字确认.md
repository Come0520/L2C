> **文档版本**：v1.0

> **状态**：✅ 已实现

---

## 1. 功能概述

签字确认是 L2C 系统中的一个**横切关注点**，它不是独立模块，而是嵌入在多个具体业务节点中的确认环节。客户通过手写签名的方式确认业务内容（报价单 / 安装完工等），签名图片上传至 OSS 存储，作为业务凭证留存。

### 1.1 核心原则

- **无独立入口**：签字确认不作为工作台的固定功能模块，而是在具体业务场景中触发
- **弹窗交互**：所有签字均通过底部弹出式弹窗（Bottom Sheet）完成，不占用整页
- **可复用组件**：签名画板封装为独立组件 `components/signature`，所有场景共用

## 2. 业务场景

### 场景一：安装现场签字（师傅端）

| 项目 | 说明 |
|------|------|
| **角色** | 安装师傅 → 客户（借用师傅手机）|
| **触发条件** | 师傅完成安装，在任务详情页点击"完工确认" |
| **流程** | 师傅上传现场照片 → 填写完工备注 → 将手机交给客户 → 客户手写签名 → 提交 |
| **入口页面** | `pages/tasks/detail/detail` |
| **API** | `POST /api/miniprogram/engineer/tasks/[id]/complete` |
| **数据存储** | `service_records.customerSignatureUrl` |

**用户故事**：
> 作为安装师傅，我需要在安装完成后让客户当场签字，确认安装质量合格，以便后续结算和避免纠纷。

### 场景二：远程签字确认（客户端）

| 项目 | 说明 |
|------|------|
| **角色** | 客户（在自己手机上操作）|
| **触发条件** | 客户收到销售/师傅发送的确认链接或通知 |
| **流程** | 客户打开链接 → 查看完工照片 → 确认无误 → 签字确认 |
| **入口页面** | `pages/tasks/customer-confirm/index` |
| **API** | `POST /api/mobile/tasks/[id]/confirm` |
| **数据存储** | `service_records.customerSignatureUrl` |

**用户故事**：
> 作为客户，安装完成后我不在现场，我需要通过手机查看师傅拍的现场照片，确认质量后远程签字。

### 场景三：报价单确认签字（客户端）

| 项目 | 说明 |
|------|------|
| **角色** | 客户 |
| **触发条件** | 销售将报价单链接发给客户，客户打开后在底部签字确认 |
| **流程** | 查看报价单明细 → 确认金额无误 → 点击"签字确认报价单" → 弹窗签字 → 提交 |
| **入口页面** | `pages/quotes/detail` |
| **API** | `POST /api/miniprogram/quotes/[id]/confirm` |
| **数据存储** | `quotes.customerSignatureUrl` + `quotes.confirmedAt` |

**用户故事**：
> 作为客户，销售发给我的报价单我需要仔细看完后签字确认，这样才能正式生效进入订单流程。

## 3. 交互设计

### 3.1 签字弹窗（通用）

```
┌──────────────────────────────┐
│  签字确认                  ✕  │
│  请在下方区域手写签名...       │
│ ┌──────────────────────────┐ │
│ │                          │ │
│ │    [签名画板 Canvas]      │ │
│ │                          │ │
│ └──────────────────────────┘ │
│  ┌──────┐  ┌──────────────┐  │
│  │ 重签  │  │   确认提交    │  │
│  └──────┘  └──────────────┘  │
└──────────────────────────────┘
```

- **弹出方式**：从底部滑出（Bottom Sheet），背景半透明遮罩
- **签名画板**：白色背景 + 虚线边框，支持高 DPI 屏幕
- **按钮**："重新签名"（左侧灰色）+ "确认提交"（右侧蓝色/绿色主色）

### 3.2 完工确认弹窗（场景一专用）

在签字弹窗基础上增加：
- 📷 **现场照片上传**（最多 6 张，支持相册 / 拍照）
- 📝 **完工备注**（选填 textarea，最多 200 字）
- ✍️ **客户签字区**（复用签名画板组件）

### 3.3 已签字状态展示

| 场景 | 展示方式 |
|------|----------|
| 报价单已签字 | 显示签名图片 + 确认时间 + "生成订单"按钮 |
| 任务已完工 | 显示现场照片 + 客户签字图 + 绿色左边框标识 |
| 客户确认页已签字 | 顶部绿色横幅 "✅ 您已完成签字确认" |

## 4. 技术实现

### 4.1 签名画板组件

| 属性 | 路径 |
|------|------|
| 组件目录 | `miniprogram/components/signature/` |
| 文件列表 | `signature.ts` / `signature.wxml` / `signature.wxss` / `signature.json` |
| 单元测试 | `__tests__/components/signature.test.ts`（3 个用例） |

**核心能力**：
- Canvas 2D API + 高 DPI 适配（`pixelRatio` 缩放）

## 5. 离线与弱网处理

系统已内置离线签名队列机制（`src/features/service/installation/logic/offline-signature.ts`）：

- 当网络不可用时，签名数据暂存本地 `localStorage`
- 网络恢复后自动重试上传
- 支持状态跟踪：`pending` → `uploading` → `uploaded` / `failed`
- 失败自动重试，最大重试次数可配

## 6. 安全措施

- **频率控制**：单用户每 3 秒最多确认 1 次（`RateLimiter`）
- **租户隔离**：所有查询和更新均携带 `tenantId` 条件
- **状态校验**：只有 `PENDING_APPROVAL` 状态才允许确认
- **审计日志**：每次签字确认均记录审计日志（`AuditService.log`）
- **Zod 输入验证**：`signatureUrl` 必须为合法 URL 格式
