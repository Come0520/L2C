## 1. 模块概述

管理后台模块负责平台级别的系统管理，包含**角色管理**、**租户设置**和**师傅管理**三个核心子模块。所有操作均需权限校验和审计日志。

## 5. 安全机制

### 5.1 通用安全措施
- **Zod 校验**：所有输入参数经 Zod Schema 严格校验
- **多租户隔离**：所有查询强制携带 `tenantId` 条件
- **速率限制**：`AdminRateLimiter` 限制管理操作频率
- **审计日志**：所有写操作均通过 `AuditService.log` 记录

### 5.2 特殊安全策略
- **系统角色保护**：`isSystem=true` 的角色不可修改或删除
- **权限提升防护**：验证分配的权限在合法权限白名单内
- **自禁保护**：管理员不能禁用自己的账号
- **敏感字段排除**：查询结果不返回密码哈希等敏感数据

### 5.3 策略引擎（Policy Engine）
- 支持 ABAC 扩展（基于属性的访问控制）
- "一票否决制"：任一规则拒绝则整体拒绝
- 内置工作时间策略示例（23:00-06:00 限制高危操作）

## 6. Zod Schema 校验规则一览

### 角色管理
- `name`：1-50 字符，仅中文/英文/数字/空格/下划线
- `code`：2-30 字符，仅小写字母/数字/下划线（可选，默认从 name 生成）
- `permissions`：至少包含一个权限字符串
- `roleId`：UUID 格式

### 租户设置
- `name`：1-100 字符
- `logoUrl`：有效 URL 或 null
- `applicantPhone`：手机号正则 `/^1\d{10}$/`
- `applicantEmail`：有效邮箱格式
- `mfa.method`：枚举 `totp` / `sms`

### 师傅管理
- `id`：UUID 格式（防路径注入）
- `name`：1-50 字符
- `phone`：`/^1\d{10}$/` 正则校验
- `avatarUrl`：有效 URL 或空字符串
- `pageSize`：1-100（防 DoS）
- `search`：最长 100 字符
