# UX/UI 差距分析 (Gap Analysis)

基于最新设计的工作流 (`business_flows.md`) 与现有 UI/UX 规范 (`ui_ux_requirements.md`) 的对比，识别出以下关键缺失的界面设计需求。

## 1. 测量服务 (Measurement)

**缺失点**: 现有规范仅提到“移动端优先”，缺乏对 **三种测量场景 (Scenarios)** 的差异化设计。

**新增 UX 需求**:
*   **场景一 (方案验证)**:
    *   **界面**: 需展示“预选方案列表”，支持逐项勾选“可行/不可行”。
    *   **交互**: 针对“不可行”的项，点击即弹出“调整方案”表单。
    *   **反馈**: 必须有“一键引用”功能，将预选方案数据复制到实测数据中。
*   **场景二 (盲测)**:
    *   **界面**: 空白采集单模式，需支持快速添加“空间”和“窗户”。
    *   **交互**: 拍照组件应置顶，鼓励先拍照后填数据。
*   **场景三 (自测)**:
    *   **界面**: 增加“费用预估”填写卡片。
    *   **审批**: 提交后显示审批状态进度条。

## 2. 订单履约 (Order Fulfillment)

**缺失点**: 缺乏 **智能拆单 (Smart Split)** 的可视化交互设计。

**新增 UX 需求**:
*   **拆单预览页**:
    *   **布局**: 左右分栏或分组列表。
    *   **展示**: 清晰展示“成品采购单”、“面料采购单”、“内部备货单”的分组结果。
    *   **交互**: 支持**拖拽 (Drag & Drop)** 商品在不同采购单/供应商之间移动（用于人工纠偏）。
    *   **高亮**: 自动拆单结果应有标记 (如 "Auto-Split")，人工修改后变色。

## 3. 供应链与采购 (Supply Chain)

**缺失点**: 缺乏 **双轨采购 (Dual Routes)** 和 **木桶效应 (Barrel Effect)** 的直观展示。

**新增 UX 需求**:
*   **订单详情页 - 采购进度**:
    *   **可视化**: 使用“进度管道图”或“甘特图”展示并行采购进度。
    *   **木桶效应提示**: 当部分采购单完成但订单未流转时，需显式提示“等待 XX 采购单完成”。
*   **面料入库与加工**:
    *   **关联跳转**: 采购单详情页需有明显入口跳转至生成的“加工单”。
    *   **状态同步**: 加工单状态变化应实时反馈在采购单/订单的时间轴上。

## 4. 财务结算 (Finance)

**缺失点**: 缺乏 **佣金结算** 的透明化展示。

**新增 UX 需求**:
*   **佣金明细弹窗**:
    *   **公式展示**: 显示具体的计算公式（如 `10000 * 10% = 1000`）。
    *   **依据溯源**: 点击计算基数（如订单金额）可跳转至对应订单。

## 5. 通用交互 (General Interaction)

**缺失点**: 复杂表单的 **版本控制** 展示。

**新增 UX 需求**:
*   **测量/报价多版本**:
    *   **切换器**: 使用 `Tabs` 或 `Dropdown` 切换 V1, V2 版本。
    *   **差异高亮 (Diff View)**: 在对比模式下，高亮显示字段值的变化（参考 Git Diff UI）。
