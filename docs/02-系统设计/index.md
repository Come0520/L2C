# 系统设计目录结构说明

## 结构总览
- 01-需求分析：需求与验收相关文档
- 02-架构设计：总体架构、技术方案、基础设施与跨模块设计
- 03-模块设计：各业务模块的详细设计文档
- 04-接口设计：统一接口与协议设计
- _archive：历史文档与被合并的旧版本，保留追溯

## 排序与置顶
- 同层级文档按系统设计流程排序：需求分析 → 架构设计 → 模块设计 → 接口设计
- 同层级内按修改日期降序排列（编辑器与VCS显示为准）
- 关键设计文档：置顶为“01-软件架构设计.md”“04-技术架构设计方案.md”“03-API接口设计.md”

## 命名规范
- 统一使用 kebab-case 文件命名（与项目规范一致），保留原文件名以减少引用风险
- 版本信息放入文档头部“版本”区段或正文“变更记录”，不在文件名中增加下划线版本号
