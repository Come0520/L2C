# Task 4: Products(商品)模块 L4 升级指南

> 专属 Subagent：负责 `src/features/products` 目录

## 背景

当前 products 模块成熟度为 L3 (6.8)，主要扣分项在类型安全（仍有 4 处 `any`）、测试覆盖度（缺乏关键的分类/图片管理用例）以及前台图片展示的性能问题。本任务目标是补齐这些短板，使其达到 L4（安全与结构化级）。

## 严格工作范围

- **仅限修改**：`src/features/products/`
- **绝对禁止修改**：任何其他模块（哪怕是看起来有引用关系的）、`src/shared/`、全局数据库 Schema 等。如果发现必须修改这些地方才能完成任务，请必须停下向主线程抛错。

## 详细任务

### 1. 彻底类型清理（D2 提升）
- 搜索并定位 products 模块下的 4 处 `any`（可能是 `as any` 或 `: any`）。
- 结合 Schema 和逻辑，把它们替换为正确的具体类型。如果无法断言，退而求其次使用 `unknown` 并补充类型守卫。
- 操作完成后，保证当前目录下的 `npx tsc --noEmit` 抛出的此模块类错误降为 0。

### 2. 测试覆盖度补强（D3 提升）
- 现有测试通常围绕业务 CRUD。请重点检查 `products` 下对于**商品分类**（Category）和**图片管理**相关的核心动作（Actions）。
- 编写或补充对应的单元/集成测试。增加至少 3 个有效核心用例。

### 3. 组件性能及可用性优化（D8/D5 提升）
- 定位主要的商品列表/卡片组件。
- 请检查 Next.js `Image` 组件的使用是否恰当：是否正确配置了 `sizes`，是否加上了 `lazy` 或关键渲染首图的 `priority` 标签。

## 完工自检清单

- [ ] 本模块内再无 `any` 和 `@ts-ignore`。
- [ ] 测试运行通过：`npx vitest run src/features/products`。
- [ ] 类型检查通过：`npx tsc --noEmit` 没有任何与商品模块相关的因你的修改新引入的错误。
- [ ] 未越权修改非 `products` 模块文件。
