# Task 5: Approval(审批)模块 L4 升级指南

> 专属 Subagent：负责 `src/features/approval` 目录

## 背景

当前 approval 模块成熟度为 L3 (6.6)，作为贯穿报价、财务、折扣的核心流程控制模块，其测试覆盖不足，且 UI 交互（进度面板）和查询性能未能达到 L4 标准。

## 严格工作范围

- **仅限修改**：`src/features/approval/`
- **绝对禁止修改**：任何其他业务模块、`src/shared/`、全局数据库 Schema 等。

## 详细任务

### 1. 全链条测试闭环构建（D3 提升）
- 查阅 `approval` 的核心 Actions（发审、驳回、通过、撤销）。
- 建立全生命周期的系统测试脚本，包含：创建审批实例 -> 多级审批节点流转 -> 模拟被驳回 -> 重新提交 -> 最终通过。
- 至少编写 6 个核心交互/流转用例，确保涵盖边界条件。

### 2. 审批进度可视化 UI 完善（D5 提升）
- 优化核心展示组件（如 `ApprovalTimeline` 或进度面板）。
- 确保节点状态（Pending, Approved, Rejected）能可视化明显区分，并合理展示当前所在的责任人。

### 3. 性能优化（D8 提升）
- 审查 approval 核心列表查询和 count 查询逻辑。
- 在不需要强实时锁或者不需要全量关系图的地方使用局部更新、限制联表嵌套层数。有必要时提供优化建议（如果需要加索引，可写入提案供主线程合并）。

## 完工自检清单

- [ ] 生命周期测试 ≥ 6 个且 100% 通行（`npx vitest run src/features/approval`）。
- [ ] UI 逻辑安全，未打乱既有的客户端路由。
- [ ] 类型检查通过：`npx tsc --noEmit` 未报错。
- [ ] 未修改非 approval 核心目录的文件。
