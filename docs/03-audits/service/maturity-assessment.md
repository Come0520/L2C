# 服务与调度模块成熟度评估报告 (Service & Dispatch Module Maturity Assessment)

## 1. 模块概览

服务与调度模块是 L2C 系统在交付环节的“指挥中枢”。它涵盖了从订单自动拆分为安装任务、基于多维度加权算法的智能派单、安装师傅的移动端履约以及复杂的冲突检测机制。该模块的设计目标是最大化作业效率，同时通过技术手段（如 GPS 签到、离线签名）确保服务质量的可追溯性。

## 3. 核心设计亮点

1. **100 分制智能打分引擎 (Scoring Engine)**：综合技能匹配 (50%)、工作负载 (30%)、服务评价 (20%) 以及地理距离和时间可用度。算法逻辑清晰，且支持根据业务权重动态调整。
2. **多级冲突检测 (Multi-level Conflict Detection)**：区分“硬冲突”（禁止操作）和“软冲突”（风险预警）。软冲突通过对赶场距离 (>50km) 和时间间隔 (<2h) 的交叉判定，模拟了真实的现场作业复杂性。
3. **离线优先 (Offline-First) 签名机制**：针对工地、新房等弱网场景，系统通过 `localStorage` 暂存签名，并配合自动同步和重试机制，确保了完工存证的 100% 到达。
4. **基于品类的任务自动拆分 (Smart Split)**：系统能自动将包含窗帘、墙布、成品家具的复杂订单拆分为具有不同技能要求的子任务，并推荐对应技能的师傅，大幅降低了人工排单难度。

## 4. 改进建议

- **路况介入**：目前距离计算主要基于直线距离，未来可引入第三方地图 API 的实时路况数据，进一步优化“赶场风险”判定。
- **动态排班可视化**：可以增加甘特图形式的师傅排班视图，使调度员能更直观地看到全城师傅的实时负载。
