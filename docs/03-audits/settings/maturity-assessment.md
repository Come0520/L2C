# 系统设置模块成熟度评估报告 (Settings Module Maturity Assessment)

## 1. 模块概述 (Overview)

系统设置模块（Settings）是 L2C 平台的多租户配置大脑。它不仅管理基础的用户与角色，还通过动态参数体系支撑了全系统的业务逻辑配置（如财务核销规则、线索回收天数等）。

- **评估分数**: L5 (卓越级)
- **核心逻辑**: RBAC 权限模型、动态系统参数 (System Settings)、租户初始化、安全审计与 Magic Link。

## 2. 评分维度 (Scoring)

| 维度           | 评分 | 关键证据                                                                          |
| :------------- | :--- | :-------------------------------------------------------------------------------- |
| **功能完备性** | 5.0  | 包含用户、角色、渠道、全局参数、审批流配置、提醒规则。                            |
| **安全性**     | 5.0  | 闭环保护逻辑：禁止删除自己、禁止禁用最后一个管理员、禁止删除有用户的角色。        |
| **自动化程度** | 5.0  | 租户创建时自动同步系统角色（syncSystemRoles）并初始化默认设置。                   |
| **可扩展性**   | 5.0  | 动态参数体系支持多种类型（JSON/ENUM等），无需修改数据库 Schema 即可增加业务开关。 |
| **易用性**     | 4.5  | 提供 Magic Link 帮助管理员快速协助下属重置访问；支持批量更新设置。                |
| **审计与追踪** | 5.0  | 所有敏感配置均有 `systemSettingsHistory` 留存，精确到字段级的 Audit Log。         |

## 3. 技术亮点 (Technical Highlights)

### 3.1 动态参数体系 (`system-settings-actions.ts`)

- **强类型校验**: 存储为字符串/JSON，但通过 `valueType` 字段在应用层强制转换，兼具灵活性和类型安全。
- **历史记录**: 每次 `updateSetting` 都会在 `systemSettingsHistory` 中记录 “谁、在什么时候、将什么键、从旧值改为了新值”。

### 3.2 鲁棒的 RBAC 模型 (`roles-management.ts`)

- **系统/自定义分离**: 系统预置角色受 `isSystem` 保护，其权限定义在代码中维护，数据库仅同步，防止误删或篡改核心权限。
- **并发与依赖检查**: 在删除角色前执行 SQL 检查：`roles @> JSON.stringify([role.code])`，确保无引用删除。

### 3.3 高安全 Magic Link 实现 (`user-actions.ts`)

`generateUserMagicLink` 提供了企业级的密码重置替代方案：

- **同租户校验**: 严格限制管理员只能操作本租户用户。
- **状态撤销**: 生成新链接前自动作废旧链接。
- **审计留痕**: `MAGIC_LINK_GENERATED` 动作被记录，防止管理员滥用权限。

### 3.4 渠道层级管理 (`actions.ts`)

- 支持无限极（或多级）渠道分类，通过 `parentId` 实现树形结构。
- 具备递归删除保护，防止产生孤儿数据。

## 4. 改进建议 (Recommendations)

1. **多级管理员**: 目前主要区分 ADMIN/OWNER，对于大型企业租户，可能需要“部门管理员”权限，目前主要靠自定义角色实现，可进一步产品化。
2. **配置灰度**: 核心业务开关（如分单规则）如需变更，目前是立即生效，未来可考虑增加“预生效”或“定时生效”功能。
3. **前端权限 Mock**: 在复杂权限测试时，前端需要更便捷的方式 Mock 权限集合以验证 UI。
