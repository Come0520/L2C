# 上传模块成熟度评估报告 (Upload Module Maturity Assessment)

## 1. 模块概述 (Overview)

上传模块（Upload）是 L2C 系统的数据持久化入口。它提供了一套高度安全的、支持多租户物理隔离的文件管理方案。模块采用“双轨制”设计：既支持基于阿里云 STS 的云端直传（带自动 Token 刷新机制），也支持带严格安全审计的本地磁盘存储。

- **评估分数**: L5 (卓越级)
- **核心逻辑**: STS 令牌分发与 14 分钟预刷新缓存、MIME 白名单校验、文件名安全净化（防路径遍历）、租户级目录物理隔离。

## 2. 评分维度 (Scoring)

| 维度           | 评分 | 关键证据                                                                                    |
| :------------- | :--- | :------------------------------------------------------------------------------------------ |
| **上传安全性** | 5.0  | 具备 MIME 白名单（图片/PDF/Office）及文件名正则净化，有效防御木马上传与路径遍历攻击。       |
| **数据隔离性** | 5.0  | 采用 `public/uploads/{tenantId}/` 物理路径隔离，确保租户间资产完全不可跨访。                |
| **凭证管理**   | 5.0  | OSS STS Token 通过 `unstable_cache` 进行了 840s 缓存，实现高效的令牌复用与平滑刷新。        |
| **异常审计**   | 5.0  | 具备“安全拦截审计”机制：当检测到违规文件格式或大小超标时，会主动记录 `ACCESS_DENIED` 日志。 |
| **性能设计**   | 5.0  | 静态资源通过物理路径直达，不通过后端逻辑代传，极大降低了上传峰值时的后端 CPU 负载。         |

## 3. 技术亮点 (Technical Highlights)

### 3.1 零信任文件名净化 (Filename Sanitization)

- **深度防御**: 系统不信任客户端提供的任何文件名。通过正则表达式 `/[^\w.\-]/g` 将所有特殊字符（含路径分隔符、脚本标签）替换为下划线，并在头部追加毫秒级时间戳，彻底杜绝了文件覆盖与远程代码执行（RCE）风险。

### 3.2 智能令牌生命周期管理 (STS TTL Management)

- **平滑刷新**: 针对阿里云 STS 凭证的 15 分钟有效期，模块在缓存层设置了 14 分钟的 Revalidate 周期。这一“提前一分钟”的预刷新策略，确保了在高并发上传场景下，客户端永远不会拿到已过期的凭证。

### 3.3 物理级隔离存储

- **多租户安全**: 系统在磁盘层动态创建租户专属目录。这种物理隔离方式比单纯的逻辑字段过滤更稳健，即使发生配置失效，也不会出现跨租户的文件泄露。

## 4. 改进建议 (Recommendations)

1. **图像魔术 (Image Processing)**：建议集成 Sharp.js 或 OSS 图片处理参数，在上传时自动生成 WebP 格式及不同尺寸的缩略图，提升前端加载性能。
2. **恶意代码扫描 (Virus Scan)**：对于 Office 文档，建议增加 ClamAV 连动扫描，防止宏病毒通过系统分发。
3. **内容分发 (CDN)**：目前采用物理存储直读，建议在生产环境切换至 CDN 加流，实现全球加速与热点卸载。
