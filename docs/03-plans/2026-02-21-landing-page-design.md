> **版本**：v1.1（评审后修订版）

> **作者**：产品团队
> **状态**：需求已确认

## 1. 项目概述

### 1.1 背景

L2C 是一款面向**家具设计/窗帘行业**的 Lead-to-Cash 全流程 SaaS 系统，已具备完整的后台管理能力（CRM、报价、订单、供应链、财务、云展厅等）。

**当前问题**：系统没有公开的品牌官网，访客进入根路径 `/` 后直接跳转到登录页面，缺少品牌故事讲述、产品价值传递和获客转化入口。

### 1.2 项目目标

为 L2C 打造一个**awwwards 级别**的品牌官网主页，在进入 SaaS 系统登录之前，让潜在客户：
1. **了解我们**：知道 L2C 是谁、为什么做、能帮助谁
2. **感受痛点共鸣**：意识到自身业务痛点，产生"我也需要"的冲动
3. **理解产品价值**：清晰看到 L2C 的核心功能和差异化优势
4. **统一入口**：作为现有 SaaS 系统的统一登录/注册入口
5. **动态运营**：提供广告位展示最新活动，并支持通过后台灵活管理落地页内容
6. **最终转化**：注册或联系我们

### 1.3 目标用户

| 角色 | 描述 | 关注点 |
|------|------|--------|
| **窗帘/家具门店老板** | 中小型门店经营者 | 经营效率、成本控制、人员管理 |
| **销售经理** | 负责团队 KPI | 线索转化率、报价效率、团队协作 |
| **行业从业者** | 测量师、安装师、设计师 | 工具易用性、移动办公、方案协同 |

### 1.4 设计原则

> 本页面定位为**沉浸式大片级体验**，用户从上到下完整观看品牌故事，不设浮动 CTA 或弹窗干扰。

## 2. 设计规范

### 2.1 交互风格

**纵向滚动叙事 (Scroll Storytelling)**
- 用户从上往下滚动，像观看一部影片一样阅读产品故事
- 每个"幕"展示一个核心信息，GSAP ScrollTrigger 驱动动画
- 参考：Apple 产品页、Stripe 官网

### 2.2 视觉调性

**现代极简 (Modern Minimal)**
- 大面积留白，让产品截图和动画成为视觉焦点
- 品牌色通过 CSS 变量预留，后期可通过 `globals.css` 整体调整
- 强调简洁的排版层次感

### 2.3 设计令牌 (Design Tokens)

```css
:root {
  /* 品牌色 - 预留，后期统一调整 */
  --brand-primary: #2563eb;       /* 主品牌色 - 待定 */
  --brand-primary-light: #3b82f6; /* 品牌色浅色变体 */
  --brand-accent: #f59e0b;        /* 强调色 - 待定 */

```

### 第 1 幕：品牌故事与运营广告 (Hero Section)

> **核心问题**：我们是谁？能帮助谁？最近有什么新动态？

首屏采用融合布局，不仅包含品牌信息，还集成动态运营位。

| 属性 | 内容 |
|------|------|
| **布局** | 左侧品牌叙事 + 右侧/底部广告轮播，或者全屏轮播（图文结合） |
| **主标题** | 品牌 Slogan（示例："让窗帘生意，回归简单"） |
| **副标题** | 一段 2-3 行的创始故事/品牌宣言 |
| **主 CTA** | "免费开始" 按钮 -> `/register` |
| **运营轮播** | **[CMS] 广告/活动推广轮播区 (Carousel)**，展示最新活动、系统更新或限时优惠 |
| **背景** | 极简白/浅灰 + 可选品牌装饰图形 |

**动画需求**：
- 文字逐行 fadeIn + translateY（向上浮动入场）
- 广告轮播支持平滑的淡入淡出 (Fade) 或滑动 (Slide) 切换，带有进度指示器
- 向下滚动时：标题缩小并渐隐，平滑过渡到第 2 幕

**文案参考**（最终文案待定）：

| 报价 | "改一个窗帘尺寸，整张报价单要重做" |
| 跟单 | "订单跟着跟着...... 就跟丢了" |
| 跟单 | "客户打电话问进度，我也不知道" |
| 跟单 | "哪个订单该发货了？翻了 20 条微信记录" |
| 测量 | "测量师到了现场不知道要测哪些窗户" |
| 测量 | "测量数据写在纸上，回来就找不到了" |
| 财务 | "供应商对账每月耗 3 天" |
| 财务 | "老板要数据，全靠 Excel 手动统计" |
| 财务 | "哪个项目赚了哪个亏了？说不清楚" |
| 管理 | "安装排期全靠微信群喊" |
| 管理 | "新员工来了，培训一个月才能上手" |
| 管理 | "三个销售同时跟了一个客户，谁都不知道" |
| 获客 | "展厅没人来，线上不知道怎么推广" |
| 获客 | "客户加了微信就再也没回过消息" |
| 协作 | "设计师、测量师、安装师信息不同步" |
| 协作 | "图纸传来传去搞不清哪个是最新版" |
| 供应链 | "布料到了才发现颜色不对" |
| 供应链 | "采购下单手抄订单信息，经常出错" |

#### 交互规格

| 状态 | 行为 |
|------|------|
| **默认** | 弹幕从右侧飘向左侧，多轨道随机分布，速度各异 |
| **鼠标悬停 (Hover)** | 该条弹幕减速停留，放大 1.05x，添加高亮边框 + 阴影 |
| **鼠标点击 (Click)** | 弹出解决方案浮层卡片 |
| **关闭浮层** | 点击 X 或卡片外区域关闭，弹幕恢复飘动 |

**解决方案浮层卡片结构**：
```
+----------------------------------+
| X (关闭按钮)                      |
|                                  |
| 痛点                              |
| "测量师到了现场不知道要测哪些窗户"   |
|                                  |
| L2C 解决方案                      |
| 订单自动生成测量任务单，包含        |
| 客户地址、窗户清单、特殊要求。      |
| 测量师打开手机即可查看完整方案。    |
|                                  |
| [功能截图/示意图]                  |
|                                  |
+----------------------------------+
```

**移动端适配**：弹幕效果保留（速度减慢），点击交互不变，浮层改为底部抽屉 (Bottom Sheet)。

#### 动画需求
- 弹幕采用 GSAP 水平 tween，无限循环
- 多轨道（4-6 条水平轨道），随机 `y` 位置和 `duration`
- 捕获动画：`scale: 1.05` + `boxShadow` 变化 + `border` 高亮
- 浮层弹出：从点击位置向中心弹性缩放 (`ease: "back.out(1.4)"`)

## 4. 后台管理系统 (CMS 系统需求)

为了让运营团队灵活掌控落地页，需在现有 SaaS 管理后台（如 `/(platform)/admin` 或专属设置模块）中增加 **"官网管理 (Website CMS)"** 模块。

### 4.1 可配置内容模块

| 模块 | 可配置项 | 说明 |
|------|----------|------|
| **顶部图文/轮播广告** | 图片/视频、主标题、副标题、按钮文案、跳转链接、排序时间 | 支持新增、启用/停用、排序控制第一幕的广告展示 |
| **痛点弹幕** | 痛点文案、匹配的解决方案文案、对应功能截图 | 可动态增删痛点条目，丰富互动内容 |
| **功能亮点** | 标题、卖点、配图、显示顺序 | 针对未来产品迭代，动态调整展示功能 |
| **客户认可** | 数字指标 + 客户评价（头像、姓名、职位、内容） | 动态更新运营数据和真实客户反馈 |

### 4.2 技术实现方案 (CMS)

- **数据库存储**：在数据库中新增表或在已有 settings 配置表中以 JSON 格式存储官网动态内容。
- **服务端渲染 (SSR/ISR)**：使用 Next.js 的 ISR (Incremental Static Regeneration) 或服务器组件 (RSC)，在后台更新内容后，主页可快速展示最新数据，不牺牲加载速度和 SEO。
- **后台页面结构**：在 L2C 设置菜单中新增一个 "官网首页设置" 大类，分 Tab 管理各区块数据。

### 5.1 路由结构

### 5.2 技术选型

| 技术 | 用途 | 说明 |
|------|------|------|
| **GSAP + ScrollTrigger** | 滚动驱动动画 | 已安装在 `node_modules` |
| **CSS Variables** | 品牌色管控 | 后期一键换肤 |
| **Framer Motion** | 组件过渡动画 | 与现有系统保持一致 |
| **next/image** | 图片优化 | 懒加载 + WebP |
| **Intersection Observer** | 触发数字计数器等 | 浏览器原生 API |

### 5.3 性能策略

- 所有动画组件使用 `'use client'` + `dynamic()` 动态导入
- 图片使用 WebP 格式 + 响应式 `srcset`
- GSAP 按需导入，仅加载 ScrollTrigger 插件
- 移动端通过 `prefers-reduced-motion` 简化动画
- 首屏 (Hero) 优先加载，后续章节延迟加载

## 6. 数据埋点规划

为了用数据驱动后续迭代和运营决策，页面需要内置完整的用户行为追踪。

### 6.1 关键事件追踪

| 事件名 | 触发时机 | 采集数据 | 业务价值 |
|--------|----------|----------|----------|
| `page_view` | 页面加载 | 来源渠道、设备类型、地域 | 流量分析 |
| `section_reach` | 每个章节滚动到视口 | 章节序号、停留时长 | 了解用户阅读深度 |
| `danmaku_click` | 点击痛点弹幕 | 痛点 ID、痛点文案 | **最核心指标**：了解用户最关心的问题 |
| `danmaku_hover` | 鼠标悬停弹幕 | 痛点 ID、悬停时长 | 辅助分析用户兴趣点 |
| `carousel_interact` | 切换广告轮播 | 广告 ID、手动/自动 | 评估广告素材效果 |
| `cta_click` | 点击任何 CTA 按钮 | 按钮位置、按钮文案 | 转化入口分析 |
| `register_start` | 跳转到注册页 | 来源章节 | 注册漏斗起点 |
| `login_click` | 点击登录按钮 | -- | 老用户回访率 |

### 6.2 分析看板

后续可在管理后台构建简单的官网数据看板：
- **流量概览**：UV / PV / 平均停留时间 / 跳出率
- **章节漏斗**：各章节到达率（第 1 幕 100% -> 第 2 幕 ?% -> ...... -> 最后一幕 ?%）
- **痛点热力图**：哪些弹幕被点击最多，指导产品和销售策略
- **转化追踪**：访问 -> 注册 的完整转化率

## 7. SEO + GEO 策略

除传统搜索引擎优化 (SEO) 外，还需考虑**生成式引擎优化 (GEO, Generative Engine Optimization)**——让 AI 搜索引擎（百度文心一言、豆包、Google AI Overview、Perplexity 等）在回答用户问题时能引用 L2C 的内容。

### 7.1 传统 SEO

| 项目 | 实施策略 |
|------|----------|
| **Title 标签** | `L2C - 窗帘/家具行业一站式管理系统 | 免费使用` |
| **Meta Description** | 包含核心关键词 + 价值主张 + CTA 引导 |
| **Heading 层级** | 每个章节使用语义化的 `h2` / `h3`，确保标题包含目标关键词 |
| **图片 Alt 标签** | 所有图片使用描述性中文 `alt` 文本 |
| **结构化数据** | 添加 JSON-LD：`Organization`、`SoftwareApplication`、`FAQPage` 等 Schema |
| **Open Graph** | 配置 `og:title`、`og:description`、`og:image`，优化微信/社交分享卡片 |
| **Sitemap** | 自动生成 `sitemap.xml` |
| **Canonical URL** | 确保主页唯一入口不被重复索引 |

### 7.2 GEO (生成式引擎优化)

AI 搜索引擎在生成回答时，倾向引用**结构清晰、事实性强、权威性高**的内容。

| 策略 | 实施方式 |
|------|----------|
| **问答式内容结构** | 在 HTML 中使用 FAQ 结构，如"什么是窗帘行业 SaaS？"-> 精确回答。即使视觉上不显示，也有利于 AI 爬虫提取 |
| **引用性数据** | 在页面中嵌入**可被引用的统计数据**（如"L2C 帮助门店报价效率提升 300%"），AI 引擎喜欢带数字的事实性陈述 |
| **行业术语覆盖** | 确保页面自然包含"窗帘管理软件""家具 ERP""报价系统""云展厅"等行业搜索词 |
| **权威信号** | 客户评价、合作案例、数字指标等社会证明内容让 AI 判定为可信来源 |
| **语义化 HTML** | 使用 `<article>`、`<section>`、`<aside>` 等语义标签，帮助 AI 理解页面结构 |
| **JSON-LD 强化** | `SoftwareApplication` Schema 中包含 `applicationCategory: "BusinessApplication"`、`offers.price: "0"` 等字段 |

### 7.3 目标关键词（初步规划）

| 关键词类型 | 示例 |
|------------|------|
| **品牌词** | L2C、L2C窗帘管理 |
| **行业词** | 窗帘管理软件、窗帘 ERP、家具门店管理系统 |
| **功能词** | 窗帘报价系统、窗帘订单管理、云展厅 |
| **痛点词** | 窗帘报价怎么算、窗帘门店如何数字化 |
| **长尾词** | 免费窗帘管理软件、窗帘店老板用什么系统 |
