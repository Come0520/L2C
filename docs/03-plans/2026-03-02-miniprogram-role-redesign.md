# L2C 小程序四大角色架构设计文档

> **状态**：已审批 ✅ | **日期**：2026-03-02 | **作者**：产品设计

## 背景与目标

将小程序从功能堆砌转变为**角色聚焦**的专属工具。每个角色只看到与自己强相关的内容，减少认知负担，提升移动端效率。

---

## 角色定义与 TabBar 结构

### 全局 tabBar.list（主包 5 页，命中微信上限）

| 槽位 | 页面路径                | 角色可见        |
| ---- | ----------------------- | --------------- |
| 0    | `pages/workbench/index` | Manager, Sales  |
| 1    | `pages/leads/index`     | Sales           |
| 2    | `pages/showroom/index`  | Sales, Customer |
| 3    | `pages/tasks/index`     | Worker          |
| 4    | `pages/users/profile`   | 全部角色        |

### 各角色可见 Tab 与核心功能

#### Manager（2 Tab）

- **工作台**：审批待办卡片 + 关键指标（待审批数、本月营收、新增线索）+ 通知提醒
- **我的**：个人设置、切换 Web 端入口

> ⚠️ 复杂报表、财务数据等功能不在小程序承接，引导使用 Web 端。

#### Sales（4 Tab）

- **工作台**：日常提醒、任务待办
- **线索**：客户跟进、线索状态流转
- **云展厅**：选内容 → 配置分享权限 → 生成分享链接给客户
- **我的**：个人业绩、设置

#### Worker（2 Tab）

- **任务**：顶部 Tab 切换「待接单 / 进行中 / 已完成」，涵盖量尺和安装任务
- **我的**：收益结算（历史明细）、个人设置

> 📝 角色术语统一为 **WORKER**，与数据库枚举 `userRoleEnum` 保持一致，废弃 `installer` 旧称。

#### Customer（2 Tab）

- **展厅/首页**：浏览 Sales 分享的内容或全部展厅（依授权范围显示）
- **我的**：我的订单（含**报修**入口）、积分与 VIP 等级、收货地址

## 关键特性设计

### 1. 报价快捷编辑（Sales）

> 不占独立 Tab，嵌入报价详情页。完整报价创建建议在 Web 端操作。

Sales 打开报价详情页，底部显示「快捷编辑」按钮（仅 Sales 角色可见）。进入编辑模式后：

| 操作                              | 版本  | 难度        |
| --------------------------------- | ----- | ----------- |
| 改单价 / 改数量                   | v1 ✅ | ⭐ 简单     |
| 替换商品（复用 product-selector） | v1 ✅ | ⭐⭐ 中等   |
| 删除商品                          | v1 ✅ | ⭐ 简单     |
| 新增商品（需选房间+商品+尺寸）    | v2 ⏳ | ⭐⭐⭐ 复杂 |

保存后触发重新计算总价并同步服务端。

### 2. 云展厅分享权限模型（Sales → Customer）

Sales 分享展厅时，弹出**两步配置**：

**步骤一：内容范围**

- 选项 A：仅分享我选中的内容（文章/案例/商品集合）
- 选项 B：开放全部展厅内容给该客户

**步骤二：二次传播权限**

- 🚫 不允许客户再分享（私密链接）
- ✅ 允许客户分享（裂变传播）

**后端数据模型**（新增 `showroom_shares` 表）：

```
shareId      - 唯一 token（URL 参数）
salesId      - 分享者（Sales 用户 ID）
contentIds   - 授权内容 ID 列表（空 = 全部展厅）
allowReshare - 是否允许二次分享 (boolean)
customerId   - 绑定客户 ID（可选，便于追踪）
expiresAt    - 有效期
```

Customer 打开链接时，根据 `shareId` 判断：展示范围 + 是否显示「转发分享」按钮。

### 3. 任务页内部结构（Worker）

顶部 Tab 切换：

- **待接单**：新派单，支持滑动接受
- **进行中**：今日/本周排期（按日历视图）
- **已完成**：历史记录（含收益明细入口）

### 4. 报修入口（Customer）

路径：「我的 → 我的订单 → 订单详情」

- 符合条件的订单（状态为「已完工」）底部显示「申请报修」按钮
- 进入报修流程后，走现有的 After-Sales 服务模块

### 5. 消息通知（Customer）

走**微信订阅消息**（服务通知），触发场景：

- 订单状态变化（发货、安装完成等）
- 报修单状态更新（已受理、已安排上门等）
