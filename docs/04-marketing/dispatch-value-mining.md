# dispatch (调度与全域履约中心) 模块价值挖掘与卖点报告

> 挖掘日期：2026-02-23
> 目标模块：`src/features/dispatch/`

## 🎯 核心定位 (One-Liner)
**L2C 的最强中央外卖大脑：通过算法与地图定位，将送货、安装、售后资源 100% 榨干，不跑冤枉路，不留爽约单！**

## 💎 核心卖点清单 (Killer Features)

### 1. 智能路线排程与 LBS 地理推荐 (Map Routing)
*   **技术支撑**: 基于地理位置的长短距计算，结合 `scheduleAt` 和 `installationStatus`。
*   **企业价值**: 调度不再依赖内勤大姐的经验。系统自动向调度员推荐最优线路，让一个配送安装组一天能多跑两家客户。
*   **降本价值**: 直接省油费、省人工！最大化挤干外勤团队的产能。
*   **话术金句**: “就像给你的施工队装上了滴滴大脑，接单不绕路，每天多干两家活。”

### 2. 人效监控与满载率看板 (Utilization Matrix)
*   **技术支撑**: 透视每一个师傅当天的日历排期，显示每天的“满载度 (Load Factor)”。
*   **管理价值**: 解决传统外包施工团队“今天撑死，明天饿死”的管理难题。老板直观看到谁在偷懒，谁是真正的劳模。
*   **话术金句**: “外勤的工资没白发：系统替你死盯每一分钟的劳动产出率。”

### 3. 一键甩单与外部师傅协同 (Outsource Dispatch)
*   **技术支撑**: 将订单通过 `dispatchType = 'outsourced'` 剥离给外部众包师傅，带独立结款核算机制。
*   **老板价值**: 旺季订单井喷不用怕，不用养一堆发不起基本工资的闲人。一键将超配订单甩给外部散客安装工。
*   **话术金句**: “养兵千日太贵，旺季一键扩军！用 L2C 轻资产玩转全域交付。”

## 🎬 场景化故事 (User Stories)

### 场景一：根治“放客户鸽子”导致的超级客诉
*   **痛点**: 遇到下雨或上一家安装拖胶了，师傅迟到大半天，客户在空房里暴跳如雷。调度员只能给师傅打电话，然后又安抚客户，两头受气。
*   **解决方案**: L2C 智能调度屏监控实时订单流转状态。当系统判定上一单卡点超时，直接在群内触发红灯警告。调度中心可一键紧急改派距离客户仅 3 公里的备选空闲师傅。
*   **结果**: 化解服务危机于无形。客户等来的不再是扯皮，而是系统发送的“您的安装工已就位”的超级短信。
