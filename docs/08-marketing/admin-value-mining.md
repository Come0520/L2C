# admin (后台配置中枢) 模块价值挖掘与卖点报告

> 挖掘日期：2026-02-23
> 目标模块：`src/features/admin/`

## 🎯 核心定位 (One-Liner)

**L2C 的中央变速箱：在不写一行代码的情况下，用一套超级开关让一家装企的业务规则和组织架构瞬间变形。**

## 💎 核心卖点清单 (Killer Features)

### 1. 细胞级权限原子控制 (Atomic RBAC System)

- **技术支撑**: 基于精细化的 Role-Based Access Control 矩阵，将所有操作按钮（如：看底价、批打折、删客户）打散为独立开关。
- **老板风控价值**: 彻底告别人脑判别权限的安全隐患！员工离职瞬间一键剥夺所有系统权限，“一秒钟断网”，甚至连手机上之前缓存的数据全部强行过期。
- **话术金句**: “谁能看多深的账，谁能批多大的折，全在您弹指间的按键掌控。”

### 2. 积木化组织架构与数据隔离 (Multi-Store Data Isolation)

- **技术支撑**: 适配从单店到“总部-大区-门店-小组”的无限制嵌套层级树。
- **集团扩张价值**: 老板新开一家分店，不需要再买一套系统。在后台新建一个分支，数据自动实现完美隔离：A店的店长绝对看不到B店的客户，而老板在云端一览无余。
- **话术金句**: “你开分店的速度，就是 L2C 后台点两下鼠标的速度。”

### 3. 热插拔式业务工作流 (Hot-Swappable Workflows)

- **技术支撑**: 定义可插拔的业务状态机规则，包括退单需验证几层审批。
- **应对市场变化**: 如果今年主抓毛利，就在后台一键开启“全员工本核算硬管控”模块；如果下半年为了冲流水，一键放量前端接单容错率。企业不用被软件绑架，反过来通过软件重新定义管理标准。
- **话术金句**: “不是你削足适履适应软件，而是系统像泥石流一样贴合你的百变业务。”

## 🎬 场景化故事 (User Stories)

### 场景一：阻止“内鬼离职”的灾难级破坏

- **痛点**: 一个核心的销售主管带着不满离职了，老板最怕的是他在交接期间偷偷把系统里 500 个高意向客户的电话全部导出来卖给竞品，或者把历史订单的价格全部恶意篡改！
- **解决方案**: 老板只需在 L2C Admin 面板找到该员工，一键切换至 `Disabled` 状态。
- **结果**: 该员工的手机端和电脑端将立刻弹回登录界面并永久封禁！别说导出客户，他连看一眼昨天的聊天记录都不可能。真正的防盗抢核武器。
