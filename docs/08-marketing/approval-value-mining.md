# approval (审批流控制) 模块价值挖掘与卖点报告

> 挖掘日期：2026-02-23
> 目标模块：`src/features/approval/`

## 🎯 核心定位 (One-Liner)

**L2C 的红绿灯中枢：告别微信群里喊“老板批一下”，把所有越线操作和资金流转套在无死角的电子枷锁里。**

## 💎 核心卖点清单 (Killer Features)

### 1. 动态规则驱动的全场景审批网 (Pervasive Dynamic Approval)

- **技术支撑**: 基于状态机在 `quotes`, `orders`, `finance` 等各个高危动作中埋点拦截。
- **老板价值**: 销售打折超底线了？要给客户办理退款退单拿走公司的钱了？系统直接“喀嚓”锁死，强行触发线上审批流。老板不点头，谁也摁不动确认键。
- **管理价值**: 收权的同时不拖拉。审批全部线上留痕，时间戳精确到秒，再也不用抱着纸质单据在一群人之间踢皮球。
- **话术金句**: “看不见的权力网：规矩就写在代码里，没有特例，没有越界。”

### 2. 即时触发与多层级链式处理 (Tiered Escalation)

- **技术支撑**: 灵活的流转规则，可定义店长、财务总监、总经理多级会签与流转。
- **效率价值**: 一单 5 万块钱的退单，先推给店长，再推给区域总监，最后送达总经理手机。谁卡在哪一环，红灯亮在谁头上，秒级提醒催办。
- **话术金句**: “不是你等老板审批，是系统推着节点向前走，让卡壳无所遁形。”

### 3. 抗抵赖数字签名与永久溯源 (Non-Repudiation Audit)

- **技术支撑**: `audit_logs` 结合审批流的每一步 `action` 严格入库。
- **风控价值**: 出了亏损死不认账？翻开系统，某年某月某分某秒，谁发起的折让，谁点的同意，数字证据清晰可见。
- **话术金句**: “数字签押不背锅：每一分流走的利润，都有白纸黑字的责任人。”

## 🎬 场景化故事 (User Stories)

### 场景一：微信群“假口头指示”的终结

- **痛点**: 业务员在群里发了个截图：“老板，有个大客要退单 3 万，急死了先过了吧”。老板在开车碰巧发了个“[机智]”，财务以为是同意了，直接把钱放飞了。最后对峙，老板说不是这个意思，满地鸡毛。
- **解决方案**: L2C 废掉口头承诺！任何财务出款动作，如果没有走完系统里的标准化审批流、没有点击绿色的【同意】按钮，出纳的财务放款页面根本就没有入口。
- **结果**: 管理动作从“主观情绪”彻底变成了“客观代码”，规矩大过天。
