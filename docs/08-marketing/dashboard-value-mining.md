# dashboard (工作台) 模块价值挖掘与卖点报告

> 挖掘日期：2026-02-23
> 目标模块：`src/features/dashboard/`

## 🎯 核心定位 (One-Liner)
**L2C 的千人千面指挥舱：让销售聚焦线索，让管理层看透全局，全员效率从点开系统的第一秒开始飙升。**

## 💎 核心卖点清单 (Killer Features)

### 1. 基于角色的智能任务分发 (Role-Based Dynamic Cards)
*   **技术支撑**: 动态组件加载机制，根据用户 `role` 渲染首屏待办 (Todos) 和数据预警 (Alerts)。
*   **企业价值**: 销售一上线看到的是“今天有 5 条待回访”，老板上线看到的是“本周业绩进度 45%”，千人千面。
*   **执行价值**: 员工完全不需要在左侧菜单里到处找活干。工作台把“该你干的事”直接喂到嘴边，零培训上岗。
*   **话术金句**: “一睁眼就知道今天该干嘛，菜鸟销售也能像老手一样精准出击。”

### 2. 实时的前线战报与业绩排行榜 (Real-Time Leaderboard)
*   **技术支撑**: 基于 Redis 和 Edge Runtime 计算的高性能排名榜单。
*   **老板价值**: 营造最激烈的内部竞争氛围。团队里的“签单王”名字每分每秒在所有人屏幕上飘过，让落后分子如坐针毡。
*   **管理价值**: 无需到了月底才给业务员念紧箍咒。实时数据驱动全员自我管理。
*   **话术金句**: “不用每天开早会打鸡血，一个实时战报大屏，胜过十个主管疯狂催单！”

### 3. 一击穿透的深层数据钻取 (Deep Drill-Down)
*   **技术支撑**: 工作台所有指标卡片（Metric Cards）自带深度链接 (`link-drill-down`)，直达底层明细。
*   **业务价值**: 看到“本月作废报价单 15 份”心存疑虑？点击数字，不跳页面直接穿透到底层明细，迅速揪出问题根源。
*   **话术金句**: “看见红灯就能直接掐灭病灶，数据再也不是悬在空中的数字。”

## 🎬 场景化故事 (User Stories)

### 场景一：销售新人的第一天
*   **痛点**: 新招的销售头三天往往无所事事，不知道打给谁、处理什么。
*   **解决方案**: L2C 工作台！第一天登录，屏幕正中央赫然显示由系统自动派发的 10 条“热线索”和 3 份“待发送报价任务”。新销售跟着卡片点点点，第一天就直接进入战斗状态。
*   **结果**: 人员培训成本骤降 80%，新人首单时间缩短一半。
