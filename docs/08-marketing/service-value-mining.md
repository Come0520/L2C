# service 模块价值挖掘与卖点报告

> 挖掘日期：2026-02-23
> 目标模块：`src/features/service/`

## 🎯 核心定位 (One-Liner)

**L2C 滴滴式履约工单系统：告别派单靠吼、进度靠猜，让每一次量尺安装都像叫车一样透明准时。**

## 💎 核心卖点清单 (Killer Features)

### 1. 灵活的工单智能拆分引擎 (Smart Task Splitting)

- **技术支撑**: `task-split-actions.ts` 支持将一个大订单拆分为多个子安装任务，派发给不同的师傅或团队。
- **业务价值**: 别墅大单一天装不完？软装和定制家具的到货周期不同？L2C 支持按区域、按日期、按品类智能拆分任务单，安装服务有条不紊。
- **执行价值**: 师傅手机端只看到分配给自己的部分和对应的图纸，职责清晰，避免多团队协作时的交叉扯皮。
- **话术金句**: “大单化整为零，多工种并行不撞车，复杂的别墅大宅也能轻松交付。”

### 2. 标准化安装费计薪体系 (Installation Task Costing)

- **技术支撑**: `install-actions.ts` 中管理了每个任务的基础费用、额外增项费用和扣款逻辑。
- **老板价值**: 师傅外包多，月底结账糊涂账？每派一次单，相应的安装费用在系统中白纸黑字锁定，客户的表扬或投诉直接挂钩师傅当次奖惩。
- **管理价值**: 从此不用在微信上翻聊天记录算工资。月底一键导出，准确无误。
- **话术金句**: “干多少结多少，让财务不头疼，让好师傅更赚钱。”

### 3. 闭环的售后工单网闸 (Closed-loop Ticket Management)

- **技术支撑**: `ticket-actions.ts` 处理客户投诉、漏发补件、现场返工的完整闭环，状态可视化追踪。
- **服务价值**: 发现少发了个配件，以前容易被遗忘导致客户暴雷报警。现在现场提一个 Ticket（客诉工单），直达后端文员和仓库，问题不解决，工单亮红灯。
- **话术金句**: “把每一个售后隐患锁在系统里，不解决不闭环，把危机转化为口碑。”

## 🎬 场景化故事 (User Stories)

### 场景一：根治“师傅你到哪了”的进度焦虑

- **痛点**: 客户请了一天假在家等安装，结果上午 11 点师傅还没到，客户夺命连环 call 给销售，销售再打电话给包工头，包工头再打电话问师傅，乱作一团。
- **解决方案**: 使用 L2C。派单后，师傅在微信小程序点击“已出发”、“已到达”、“上传完工照片”和“客户电子签名”。进度就像打车软件一样实时同步在订单和客户详情页中。
- **结果**: 销售和老板不用打一个电话，客户体验拉满，好评率飙升。
