# 线索管理系统规则说明

## 1. 概述
本文档详细说明线索管理模块的系统规则，包括警告机制、逾期处理、状态流转以及权限控制，供所有销售人员参考。

---

## 2. 线索状态定义

| 状态 | 含义 | 颜色标识 |
|------|------|----------|
| 新建 | 线索首次录入，等待分配 | 蓝色 |
| 待跟进 | 已分配给销售人员，等待首次联系 | 黄色 |
| 已联系 | 已完成首次联系 | 绿色 |
| 谈判中 | 正在进行商务洽谈 | 橙色 |
| 成交 | 成功签约或完成付款 | 深绿色 |
| 失效 | 线索不再有效 | 灰色 |

## 3. 警告机制

### 3.1 警告类型一览

| 警告类型 | 触发条件 | 颜色 | 处理时限 |
|----------|----------|------|----------|
| 高意向未跟进 | 高意向线索 24 小时内无跟进记录 | 红色 | 5 分钟 |
| 预算超额 | 线索预算超过公司设定上限 | 黄色 | 30 分钟 |
| 流失风险 | 连续 3 次联系失败 | 橙色 | 1 小时 |
| 竞争对手威胁 | 检测到竞争对手信息 | 蓝色 | 2 小时 |
| 长期未更新 | 线索超过 5 天未更新状态 | 紫色 | 24 小时 |
| 重复线索 | 系统检测到可能的重复线索 | 灰色 | 48 小时 |
| 信息不完整 | 关键字段（电话/邮箱）缺失 | 黄色 | 12 小时 |
| 转化延迟 | 高意向线索超过 14 天未成交 | 红色 | 4 小时 |

### 3.2 警告处理规则
- **弹窗提醒**：警告触发后，系统弹出对应颜色的警告弹窗。
- **处理时限**：在规定时限内必须处理，否则系统自动记录为"已忽略"。
- **忽略累计**：同一线索累计忽略 3 次警告后，自动升级为"流失风险"类型。
- **审计记录**：所有警告触发和处理动作均记录审计日志。

## 4. 逾期处理规则

### 4.1 逾期阈值
| 线索类型 | 逾期阈值 | 说明 |
|----------|----------|------|
| 普通线索 | 7 天 | 超过 7 天未更新状态 |
| 高意向线索 | 3 天 | 超过 3 天未更新状态 |
| VIP 线索 | 2 天 | 超过 2 天未更新状态 |

### 4.2 逾期处理流程
1. **自动标记**：线索达到逾期阈值后，系统自动将状态变更为"失效"。
2. **列表展示**：失效线索在列表中以灰色背景显示，排序靠后。
3. **复活操作**：销售人员可在详情页点击"复活"按钮，重新启动跟进流程。
4. **归档规则**：失效线索超过 30 天未复活，自动归档至历史库。

### 4.3 逾期通知
- **销售人员**：逾期前 24 小时收到系统提醒。
- **销售负责人**：下属线索逾期后，收到汇总通知。

## 5. 状态流转规则

```
新建 → 待跟进 → 已联系 → 谈判中 → 成交
         ↓           ↓           ↓
       失效        失效        失效
```

### 5.1 状态变更权限
| 操作 | 一线销售 | 销售负责人 | 管理员 |
|------|----------|------------|--------|
| 新建 → 待跟进 | 否 | 是 | 是 |
| 待跟进 → 已联系 | 是 | 是 | 是 |
| 已联系 → 谈判中 | 是 | 是 | 是 |
| 谈判中 → 成交 | 是 | 是 | 是 |
| 任意 → 失效 | 是 | 是 | 是 |
| 失效 → 待跟进（复活） | 否 | 是 | 是 |

### 5.2 状态变更触发
- **手动变更**：用户在详情页手动更新状态。
- **自动变更**：系统根据逾期规则自动变更为"失效"。

## 8. 常见问题

### Q1: 线索被误标为失效怎么办？
销售负责人可在详情页点击"复活"按钮，重新激活线索。

### Q2: 警告弹窗没有及时处理会怎样？
系统会自动记录为"已忽略"，累计 3 次后升级为"流失风险"警告。

### Q3: 如何查看线索的操作历史？
在线索详情页底部点击"操作记录"标签页，可查看完整的审计日志。
