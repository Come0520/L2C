## 🎯 角色定位与职责

### 您的权限范围

作为**销售负责人（LEAD_SALES）**，您拥有：

**核心权限**：
- ✅ 分配线索给团队成员
- ✅ 查看团队所有线索数据
- ✅ 审批特殊申请（价格/退换货）
- ✅ 调整线索状态（处理异常）

- ✅ 设置团队目标和规则

- ✅ 查看个人工作量和业绩
- ✅ 查看跟进记录详情
- ✅ 接收超时预警
- ✅ 处理客户投诉升级

## 🚀 团队管理核心功能

### 功能 1：线索分配策略

#### 1.1 分配原则

**均衡原则**：
```
确保每位销售的线索量相对均衡
考虑：
- 当前线索存量
- 个人转化率
- 工作饱和度
```

**专长匹配**：
```
高端客户 → 经验丰富的销售
远程客户 → 远程销售能力强的成员
本地客户 → 驻店销售
```

**新人培养**：
```
优质线索：70% → 老手，30% → 新人
普通线索：50/50分配
难搞客户 → 老手带新人
```

---

#### 1.2 分配操作流程

**步骤 1：查看待分配线索**
1. 进入"线索管理"
2. 筛选状态："待分配"
3. 查看线索详情（来源/预算/紧急程度）

**步骤 2：选择合适的销售**
```
考虑因素：
✅ 当前负载（不超过20个跟踪中线索）
✅ 擅长领域（产品/客户类型）
✅ 地理位置（就近原则）
✅ 近期状态（避免分给休假/请假人员）
```

**步骤 3：执行分配**
1. 点击线索右侧操作按钮
2. 选择"分配"
3. 选择销售人员
4. 填写分配原因（可选但推荐）
5. 点击"确定"

**系统自动操作**：
- 线索状态变更为"待跟踪"
- 被分配人收到系统通知
- 启动72小时跟踪倒计时

---

#### 1.3 重新分配（转交）

**适用场景**：
```
情况1：销售请假/离职
情况2：销售负载过重
情况3：客户要求换人
情况4：销售能力不匹配
```

**操作**：
1. 找到该线索
2. 点击"重新分配"
3. 选择新的销售
4. 必须填写原因
5. 通知双方（原销售+新销售）

**⚠️ 注意事项**：
```
❌ 频繁换人会影响客户体验
❌ 不要因业绩压力强行剥夺线索
✅ 转交前与双方沟通清楚
✅ 记录详细交接内容
```

### 功能 2：团队监控仪表盘

#### 2.1 实时数据查看

**位置**：首页 → 团队概览

**核心指标**：

| 指标 | 含义 | 关注重点 |

| 本月成交 | 已完成订单 | 目标达成率 |
| 平均转化率 | 团队整体效率 | 行业对标 |
| 平均成交周期 | 从线索到成交天数 | 优化空间 |

#### 2.2 个人业绩排行

**查看路径**：团队管理 → 业绩排行

**维度**：
- 本月成交数（核心）
- 转化率（效率）
- 客均订单额（质量）
- 客户满意度（服务）

**应用场景**：
```
💡 早会通报：表扬前三名
💡 月度奖励：奖金/荣誉
💡 帮扶对象：关注倒数三名
💡 经验分享：让第一名分享经验
```

---

#### 2.3 预警中心

**8大预警类型**：

| 预警 | 含义 | 处理优先级 |
|------|------|-----------|
| 🔴 跟踪逾期 | 7天无跟进 | P0（立即） |
| 🔴 高意向无跟进 | A/B级客户长时间未联系 | P0 |
| 🟠 已报价无草签 | 报价14天未成交 | P1（当天） |
| 🟡 测量逾期 | 测量环节超时 | P1 |
| 🟡 预算超支 | 报价超预算30% | P2（3天内） |
| 🟠 流失风险 | 客户停滞 | P2 |
| 🔴 竞品威胁 | 客户比价中 | P1 |
| 🟠 无7天跟进 | 系统通用预警 | P2 |

**每日处理流程**：
```
早上第一件事：
1. 打开预警中心
2. 筛选P0级预警
3. 通知相关销售立即处理
4. 设置30分钟后检查
5. 未处理者电话督促
```

### 功能 3：审批管理

#### 3.1 价格审批

**触发场景**：
```
销售申请：
- 低于成本价
- 超过标准折扣
- 特殊优惠
```

**审批步骤**：
1. 收到审批通知
2. 查看申请详情：
   - 客户信息
   - 原价/申请价
   - 申请理由
3. 评估合理性
4. 批准/驳回（必须填写意见）

**审批原则**：
```
✅ 批准条件：
   - 大客户（订单额>5万）
   - 战略客户（长期合作价值）
   - 竞品压力（有合理依据）
   - 清库存（积压商品）

❌ 驳回条件：
   - 无充分理由
   - 低于底价
   - 频繁申请同一客户
```

---

#### 3.2 退换货审批

**申请类型**：
- 客户要求退货
- 产品质量问题
- 尺寸测量错误

**审批要点**：
```
1. 核实责任方（客户/我方/供应商）
2. 查看订单历史和沟通记录
3. 评估损失程度
4. 给出处理方案
```

**处理建议**：
```
客户责任 → 协商解决（部分承担）
我方责任 → 无条件满足
质量问题 → 供应商索赔
```

## 📊 数据驱动决策

### 分析 1：团队转化漏斗

**查看路径**：数据分析 → 转化漏斗

**典型漏斗**：
```
100个线索
   ↓ (85%)
85个开始跟踪
   ↓ (60%)
51个有效沟通
   ↓ (40%)
20个发报价单
   ↓ (50%)
10个草签
   ↓ (70%)
7个成交
```

**关键转化率**：
- **L1：分配→跟踪** = 85%（目标>80%）
- **L2：跟踪→有效沟通** = 60%（目标>55%）
- **L3：沟通→报价** = 39%（目标>35%）
- **L4：报价→草签** = 50%（目标>45%）
- **L5：草签→成交** = 70%（目标>65%）

**问题诊断**：
```
L1低 → 销售响应慢，加强时效管理
L2低 → 线索质量差或话术问题
L3低 → 需求挖掘不足
L4低 → 报价不合理或竞争力弱
L5低 → 售后服务或信任问题
```

---

### 分析 2：成交周期分析

**查看**：数据报表 → 成交周期

**指标说明**：
```
平均成交周期：从线索创建到订单完成的天数

优秀：<20天
良好：20-30天
一般：30-45天
较差：>45天
```

**优化方向**：
```
缩短周期的关键点：
✅ 快速响应（<24小时首次联系）
✅ 精准报价（一次到位）
✅ 高效测量（48小时内完成）
✅ 快速决策（减少审批环节）
```

---

### 分析 3：销售个人诊断

**工具**：销售健康度评分卡

**评分维度**（满分100）：


## 🔍 流程监控与优化

### 监控点 1：时效合规率

**关键时效节点**：

| 状态 | 时效要求 | 监控方法 |
|------|----------|----------|
| 待跟踪 | 72小时内开始 | 系统自动预警 + 日报 |
| 跟踪中 | 7天内有跟进 | 每日检查预警列表 |
| 草签 | 72小时生成报价单 | 草签后立即提醒销售 |
| 待测量 | 48小时点击安排 | 每日早会通报 |

**月度合规率目标**：>90%

**低于目标时的行动**：
```
→ 找出高频违规销售
→ 一对一谈话了解原因
→ 提供针对性培训
→ 必要时调整考核权重
```

---

### 监控点 2：流程卡点分析

**常见卡点**：

**卡点1：测量环节**
```
现象：草签后迟迟不测量
原因：
- 销售忘记点击"安排测量"
- 调度资源不足
- 客户一直约不到时间

解决：
✅ 草签后系统自动提醒
✅ 增加测量师资源
✅ 销售主动联系客户
```

**卡点2：方案确认**
```
现象：方案出来后客户拖延
原因：
- 客户犹豫不决
- 价格或风格不满意
- 决策人不在

解决：
✅ 方案前充分沟通需求
✅ 提供2-3套方案选择
✅ 设定决策截止日期
```

---

### 优化实例：缩短成交周期

**优化前**：平均35天成交

**问题诊断**：
```
环节1：线索分配延迟（2天）
环节2：首次联系延迟（3天）
环节3：测量安排延迟（5天）
环节4：方案确认延迟（7天）
```

**优化措施**：
```
✅ 线索即时分配（系统自动+人工）
✅ 24小时内必须首次联系（硬性KPI）
✅ 草签当天安排测量（流程自动化）
✅ 方案3天内确认（设截止日期）
```

**优化后**：平均23天成交 ✅ 提升34%

## 🆘 问题处理与升级

### 问题类型 1：客户投诉

#### 处理流程

**Step 1：接收投诉**
```
来源：
- 客户直接致电
- 销售上报
- 客服转接
```

**Step 2：记录详情**
```
必须记录：
✅ 客户姓名+电话
✅ 投诉内容（具体事实）
✅ 诉求（客户要什么）
✅ 涉及销售/服务人员
✅ 订单号/线索ID
```

**Step 3：初步处理**
```
紧急程度判断：
🔴 P0：安全/法律问题 → 立即上报总经理
🟠 P1：严重质量问题 → 4小时内解决
🟡 P2：一般服务问题 → 24小时内解决
```

**Step 4：解决方案**
```
标准处理：
- 道歉+解释
- 提供补救方案（换货/退款/补偿）
- 后续回访

系统操作：
→ 在客户档案记录投诉
→ 标记相关销售（影响考核）
→ 设置回访提醒
```

---

### 问题类型 2：团队冲突

**常见冲突**：

**冲突1：线索争抢**
```
场景：两个销售都认为某线索应该给自己

处理：
1. 查看系统记录（谁先联系？）
2. 询问双方诉求
3. 基于规则判断（就近/专长原则）
4. 给予未分配方补偿线索
5. 明确规则，避免再犯
```

**冲突2：业绩归属争议**
```
场景：A销售跟进，B销售成交，业绩算谁？

规则：
✅ 系统记录为准
✅ 最后下单的销售获得业绩
✅ 前期贡献者可获得辅assist奖励（20%）
```

---

### 问题类型 3：系统故障

**常见故障**：

| 故障 | 紧急度 | 处理方式 |
|------|--------|----------|

| 数据丢失 | P0 | 停止操作+IT恢复 |
| 功能异常 | P1 | 记录Bug+寻找替代方案 |

**临时应对**：
```
✅ 准备Excel线索表（备用）
✅ 关键数据每日备份
✅ 建立IT紧急联系群
```
